Lesson 10.1
Conditionals: Making Decisions in Code

Learning Objective (LO10.1)

Understand and use if, if-elif-else conditionals to make decisions in your code based on different conditions.

What Are Conditionals?

A conditional is a programming structure that allows your code to make decisions. Just like you make different choices based on circumstances ("If it's raining, I'll bring an umbrella. Otherwise, I won't."), your code can do the same thing.

Conditionals use a simple question: "Is this condition true?" If the answer is yes, the code does one thing. If the answer is no, it does something else (or nothing at all).

Think of it like a flowchart in your head: Your program encounters a decision point ‚Üí asks a yes/no question ‚Üí follows one path if true, another if false ‚Üí continues.

Key Concepts from the Video:

  • An if statement checks if a condition is true
  • If the condition is true, the code inside the if block runs
  • If the condition is false, the code is skipped
  • The condition uses comparison operators like ==, >, <, !=

Example 10.1.1: Simple If Statement

The Code:
gpa = 3.8 if gpa >= 3.7: print("Great job! You made the Dean's List!")
What Happens:

The program checks: "Is 3.8 >= 3.7?" YES, it is. So the print statement inside the if block runs.

Output: Great job! You made the Dean's List!

Adding Alternatives with If-Elif-Else

üì∫ If-Elif-Else Explained

Watch: If-Elif-Else

Duration: 4:46 minutes

üìπ Learn how to create multiple paths in your code with if-elif-else statements.

What if you want to handle multiple possibilities? That's where if-elif-else comes in:

  • if: Check the first condition
  • elif: (else if) Check another condition if the first is false
  • else: Do this if ALL previous conditions are false
Think of it like a vending machine: If the first button is pressed, do that. Else if a different button is pressed, do something else. Else (if nothing matches), return the money.

Example 10.1.2: If-Elif-Else Statement

The Code:
gpa = 3.2 if gpa >= 3.7: print("Congratulations! You made the Dean's List!") elif gpa >= 3.0: print("You're in good academic standing.") else: print("Consider meeting with an academic advisor.")
What Happens:

The program checks: "Is 3.2 >= 3.7?" NO. So it checks the elif: "Is 3.2 >= 3.0?" YES, it is!

Output: You're in good academic standing.

If We Changed gpa to 2.5:

"Is 2.5 >= 3.7?" NO. "Is 2.5 >= 3.0?" NO. So we reach the else.

Output: Consider meeting with an academic advisor.

Common Mistake: Students often forget the colon (:) at the end of if, elif, and else lines. Python requires this! Also remember: indentation matters‚Äîthe code inside the if block must be indented.

Key Terms and Definitions

Conditional
A code structure that performs different actions based on whether a condition is true or false.
If Statement
A conditional that checks if a condition is true. If it is, the code block runs. If not, it's skipped.
Elif (Else If)
Used to check additional conditions if the previous if or elif condition was false.
Else
Executed if all previous if and elif conditions are false. It's your "catch-all" option.
Comparison Operator
Symbols like ==, !=, >, <, >=, <= that compare two values and return true or false.

Lesson 10.2
Functions: Creating Reusable Code

Learning Objective (LO10.2)

Define and call functions to create reusable blocks of code that perform specific tasks.

What Are Functions?

A function is a reusable block of code that performs a specific task. Instead of writing the same code multiple times, you write it once as a function and call it whenever you need it.

Think of a function like a recipe: You write down the steps once. Then whenever you want to make that dish, you just follow the recipe instead of figuring it out from scratch every time.

Why use functions?

  • Reusability: Write once, use many times
  • Organization: Your code is cleaner and easier to understand
  • Maintenance: If you need to fix a bug, you fix it in one place
  • Testing: Easier to test small pieces of code

Understanding Functions

üì∫ Functions Explained

Watch: Functions

Duration: 4:41 minutes

üìπ Learn how to define functions, use parameters, and return values.

Parts of a Function

The Structure of a Function

def function_name(parameter1, parameter2): # Function body - the code that runs result = parameter1 + parameter2 return result

Breaking it down:

  • def: Keyword that means "define a function"
  • function_name: A name you choose for your function (use descriptive names!)
  • parameters: Variables that receive values when the function is called (optional)
  • : Colon at the end of the definition line
  • body: Indented code that runs when the function is called
  • return: Sends a value back to the code that called the function (optional)

Example 10.2.1: A Simple Function Without Parameters

Define the Function:
def greet(): print("Hello! Welcome to Python!")
Call the Function:
greet() greet() greet()
Output:
Hello! Welcome to Python! Hello! Welcome to Python! Hello! Welcome to Python!

The same message prints three times! We called the function three times, and each time it ran the code inside.

Example 10.2.2: Function With Parameters (Our Academic Standing Function!)

Define the Function:
def academic_standing(student_gpa): if student_gpa >= 3.7: return "Congratulations! Dean's List!" elif student_gpa >= 3.0: return "Good academic standing." else: return "Meet with an advisor."
Call the Function With Different Values:
message1 = academic_standing(3.8) print(message1) message2 = academic_standing(3.2) print(message2) message3 = academic_standing(2.5) print(message3)
Output:
Congratulations! Dean's List! Good academic standing. Meet with an advisor.

Each call to the function returns a different message based on the GPA passed in!

Remember: First you define a function (write what it does), then you call a function (use it). You can define a function once and call it many times!

Key Terms

Function
A reusable block of code that performs a specific task. Defined once, called many times.
Parameter
A variable listed in the function definition that receives a value when the function is called.
Argument
The actual value passed to a function when you call it. If parameters are the placeholder, arguments are the actual values.
Return Value
The value that a function sends back to the code that called it using the return keyword.
Function Call
When you use a function by typing its name followed by parentheses, like function_name()

Program Deconstruction: A Complete Example

Let's analyze this complete program step by step:

A Temperature Comfort Analyzer Program

# List of temperatures to analyze temperatures = [32, 72, 55, 95] city = "Providence" # Function to determine comfort level def get_comfort_level(temp): if temp >= 80: return "hot" elif temp >= 60: return "comfortable" elif temp >= 40: return "cool" else: return "cold" # Main program print("Weather Analysis for " + city) print("=" * 30) for temp in temperatures: comfort = get_comfort_level(temp) print(f"{temp}¬∞F is {comfort}")

Output of this program:

Weather Analysis for Providence ============================== 32¬∞F is cold 72¬∞F is comfortable 55¬∞F is cool 95¬∞F is hot

Breaking Down the Program

üìç Line 1-2: Variables

temperatures = [32, 72, 55, 95] - A list containing 4 temperature values

city = "Providence" - A string variable storing the city name

Purpose: Store the data we'll analyze

üîß Lines 4-14: Function Definition

def get_comfort_level(temp): - Defines a function that takes one parameter: temp

What it does: Uses if-elif-else to return a comfort level based on the temperature

Purpose: Reusable logic to categorize any temperature

üîÑ Lines 17-22: Loop

for temp in temperatures: - Loops through each temperature in the list

comfort = get_comfort_level(temp) - Calls the function for each temperature

Purpose: Process every temperature in the list and print results

How All the Pieces Work Together

  1. The program starts with data (temperatures list and city name)
  2. It defines a function that knows how to categorize temperatures
  3. It loops through each temperature in the list
  4. For each temperature, it calls the function to get a comfort level
  5. It prints the result in a nicely formatted way
Reading strategy: Start at the top and trace through what happens. When you see a function call, jump to the function definition to see what it does, then come back to where it was called.

Understanding Through Modification

One of the best ways to understand code is to modify it and predict what will happen. Consider:

Change 1: What if we changed the city to "Miami"?

The output would show "Weather Analysis for Miami" instead. All the temperature analysis would be identical.

Change 2: What if we added a temperature of 50¬∞F to the list?

We'd get a new line in the output: "50¬∞F is cool" because the function would classify 50¬∞F as cool (>= 40 but < 60).

Change 3: What if we added a new comfort level called "freezing" for temperatures below 32¬∞F?

We'd need to modify the function's if-elif-else statement to add another condition at the beginning or modify the final else.

What Is Vibe Coding?

Vibe Coding is when you describe what you want your program to do in plain English (or whatever language you prefer), and an AI assistant helps you write the code. It's like having a coding partner who can translate your ideas into working Python!

Real-world example: Professional developers use AI tools like GitHub Copilot, ChatGPT, and Claude every single day to help them code. This is a real skill you're learning right now!

Why does this work?

  • AI models understand both human language and programming language
  • You don't need to know all the syntax (exact rules) to get started
  • You can focus on the logic and ideas, not memorizing syntax
  • It makes programming much more accessible

Writing Good Vibe Coding Prompts

Clear prompts = Better code! The better you describe what you want, the better the AI can help you.

‚úÖ DO THIS:

  • Mention the programming concepts to use (functions, loops, conditionals, etc.)

‚ùå DON'T DO THIS:

  • Be vague ("create a program")
  • Ask for something too complex
  • Forget to mention what programming concepts to use
  • Copy-paste massive requirements without organizing them
