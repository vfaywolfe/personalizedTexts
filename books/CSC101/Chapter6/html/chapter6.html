Understanding the Internet

The Internet isn't a single thing‚Äîit's not owned by one company or controlled by one government. Instead, the Internet is a network of networks. Imagine billions of individual networks (your home WiFi, your school's network, corporate networks, etc.) all connected together through standardized protocols and shared infrastructure.

At its core, the Internet is built on a simple principle: any device that can follow the Internet's communication rules can connect and communicate with any other device that follows those same rules.

Key Components

  • Devices: Computers, phones, tablets, smart watches, IoT devices‚Äîanything with Internet capability
  • Networks: Local area networks (LANs) that connect devices in homes, schools, and offices
  • Routers: Devices that connect different networks together
  • Internet Service Providers (ISPs): Companies that connect your local network to the broader Internet
  • Protocols: Standardized rules for how devices communicate (like TCP/IP)
üí° Info: The Internet was originally designed by the U.S. Department of Defense as a resilient communication system that could survive partial failures. This decentralized design is why the Internet is so robust today‚Äîthere's no single point of failure.

Key Concepts & Terms

Network: A group of connected devices that can communicate with each other.
Protocol: A set of rules that devices follow to communicate successfully (like an agreed-upon language).
TCP/IP: The fundamental protocol suite that powers the Internet. TCP (Transmission Control Protocol) ensures data arrives correctly; IP (Internet Protocol) handles routing data between devices.
Bandwidth: The maximum rate of data transfer across a connection, usually measured in megabits per second (Mbps).

How Search Engines Work

Every second, billions of searches happen on the Internet. But how does typing a few keywords instantly return millions of relevant results? The answer involves three main processes: crawling, indexing, and ranking.

1. Crawling: Search engines use automated bots (called "spiders" or "crawlers") that continuously browse the web. These bots visit web pages, follow links, and discover new content.
2. Indexing: Once a page is crawled, the search engine stores information about it in a massive index. This index includes the page's content, keywords, links, and metadata.
3. Ranking: When you search, the engine doesn't look through the entire Internet in real-time. Instead, it looks through its index to find pages matching your keywords, then ranks them based on hundreds of factors including relevance, quality, and popularity.

Personalizing Search Results

Modern search engines don't return the same results for everyone. They personalize results based on search history, location, device type, user interests, and engagement patterns.

What Is an IP Address?

Every device connected to the Internet needs a unique identifier‚Äîlike a mailing address for your house. This identifier is called an IP address (Internet Protocol address). An IP address is a unique string of numbers separated by periods, such as 192.168.1.1 or 8.8.8.8

Two Versions of IP Addresses

IPv4 (Internet Protocol version 4)
Format: Four numbers between 0-255, separated by periods (e.g., 192.168.1.100)
Maximum addresses: About 4.3 billion
Status: Gradually running out as more devices connect

IPv6 (Internet Protocol version 6)
Format: Eight groups of hexadecimal numbers (e.g., 2001:0db8:85a3:0000:0000:8a2e:0370:7334)
Maximum addresses: About 340 undecillion
Status: Being implemented globally to provide more addresses

üí° Info: We're transitioning to IPv6 because we ran out of IPv4 addresses. With billions of devices connecting, we need vastly larger address space.

How DNS Works: The Internet's Address Book

When you type google.com into your browser, you're not using an IP address‚Äîyou're using a domain name. But computers need IP addresses to communicate. This is where DNS (Domain Name System) comes in. DNS is like the Internet's phonebook, translating human-readable domain names into IP addresses.

üîç Example 6.3.1: How DNS Works
You type www.example.com into your browser
Your browser asks your ISP's DNS server: "What IP address is example.com?"
If the ISP's DNS server doesn't know, it contacts other DNS servers up the chain
Eventually, a DNS server responds with the IP address: 93.184.216.34
Your browser now knows which IP address to contact
Your request is sent to that IP address, and the website is served to you

Breaking Data Into Packets

When you send data across the Internet‚Äîwhether it's an email, message, or video stream‚Äîit's not sent all at once. Instead, it's broken into small chunks called packets. Think of it like sending a book across the country by tearing it into pages, putting each page in an envelope with an address, and mailing them separately.

Packet Structure

Each packet contains:

  • Header: Source IP address, destination IP address, and routing information
  • Data: The actual information being transmitted (part of an email, video frame, etc.)
  • Footer: Error-checking information to ensure the packet wasn't corrupted

How Routers Work

A router reads the destination IP address in each packet and decides the best path for that packet. Think of routers as postal workers at sorting centers who read an address and decide which mail truck the envelope should go on.

Routers make forwarding decisions based on destination IP address, current network traffic, available routes, cost of different paths, and routing table information.

üîç Example 6.3.2: A Packet's Journey
You send an email from your computer (192.168.1.50)
The email is broken into packets, each labeled with your IP and the recipient's IP
The first packet reaches your home router
Your router forwards it to your ISP's network
ISP routers pass the packet through internet backbone networks
Other routers make decisions about the best current path based on traffic
Eventually, the packet reaches the recipient's ISP and then their device
Other packets from your email take potentially different routes
At the destination, all packets are reassembled into the original email

Physical Infrastructure: The Internet's Backbone

Fiber Optic Cables: Thousands of miles of cables under oceans, through deserts, and across mountains carry long-distance Internet traffic using light pulses.
Copper Cables and Coaxial Cables: Used for shorter distances and local connections, transmitting data as electrical signals.
Wireless Connections (WiFi and Cellular): Use radio waves to transmit data.
Satellites: Provide Internet access to remote areas.

‚ö†Ô∏è Warning: This physical infrastructure is vulnerable to natural disasters and attacks. A single underwater cable cut can disrupt Internet access for entire countries. The Internet's resilience comes from having multiple redundant paths.

Key Concepts & Terms

IP Address: A unique numerical identifier for each device on the Internet.
Domain Name: A human-readable address like google.com that corresponds to an IP address.
DNS (Domain Name System): The system that translates domain names into IP addresses.
Packet: A small unit of data with header, data, and footer.
Router: A device that reads destination IP addresses and decides the best path for data.
Bandwidth: The maximum amount of data that can travel through a connection per unit time.

What Is HTTP?

HTTP (HyperText Transfer Protocol) is the protocol used for transferring web pages and resources across the Internet. It's the language that web browsers and web servers speak to each other.

When you type a URL into your browser and press Enter, you're initiating an HTTP request. Your browser asks a server: "Give me the web page at this address." The server responds by sending back the HTML, images, CSS, and JavaScript files that make up the website.

The HTTP Request-Response Cycle

üîç Example 6.5.1: How You Get a Web Page
You type in a URL: You enter https://www.example.com/index.html in your browser
DNS lookup: Your browser asks DNS: "What's the IP address for example.com?"
HTTP request: Your browser sends an HTTP GET request to that IP address
Server processing: The web server receives the request and locates the file
HTTP response: The server sends back an HTTP response with status code 200 (success) and the HTML file
Browser rendering: Your browser reads the HTML file and renders it as a visual webpage
Additional requests: Your browser sees references to images, CSS, and JavaScript in the HTML
Loading resources: Your browser makes additional HTTP requests for each resource
Complete page: Once all resources are loaded, the complete webpage appears on your screen

What Is HTML?

HTML (HyperText Markup Language) is the language used to create web pages. It uses tags and elements to structure content and tell browsers how to display it. HTML is just text.

<!DOCTYPE html>
<html>
  <head>
    <title>My Website</title>
  </head>
  <body>
    <h1>Welcome to My Website</h1>
    <p>This is a paragraph.</p>
  </body>
</html>

When your browser receives this HTML, it reads the tags (like <h1>, <p>, <img>) and renders them as visual elements on your screen.

üí° Hint: You can see the HTML source code of any website by right-clicking on the page and selecting "View Page Source" or pressing Ctrl+U (Windows) or Cmd+Option+U (Mac). This is a great way to learn how websites are structured!

HTTP Status Codes

When a server responds to an HTTP request, it includes a status code indicating whether the request was successful:

  • 200 OK: Request succeeded; the server is sending the requested resource
  • 301 Moved Permanently: The resource has moved to a new location
  • 404 Not Found: The server couldn't find the requested resource
  • 500 Internal Server Error: The server encountered an error processing your request
  • 503 Service Unavailable: The server is temporarily unavailable

HTTPS: Secure HTTP

HTTPS (HTTP Secure) is HTTP with added encryption and security. When you see a padlock icon in your browser's address bar, you're using HTTPS. HTTPS encrypts data traveling between your browser and the server, so if someone intercepts packets, they won't be able to read the contents.

‚ö†Ô∏è Warning: Avoid entering sensitive information on websites that don't use HTTPS (no padlock icon). Without encryption, your data could be intercepted by anyone monitoring your Internet connection.

Key Concepts & Terms

HTTP: The protocol for transferring web pages between browsers and servers.
HTTPS: HTTP with added encryption.
HTML: The markup language for structuring web page content.
Web Server: A computer that stores HTML files and responds to HTTP requests.
Browser: Software that requests HTML files from servers and renders them.
URL (Uniform Resource Locator): The web address you type in your browser.

What Is the Digital Divide?

Despite the Internet's global reach, access is far from equal. The digital divide refers to the gap between people who have reliable access to the Internet and technology, and those who don't.

Types of Digital Divides

Geographic: Rural areas often lack Internet infrastructure while cities have broadband. In developing countries, access is concentrated in urban centers.
Economic: Internet access requires a device and monthly service plan‚Äîexpenses many low-income families cannot afford (\$50-80/month).
Age: Older adults are less likely to have grown up with computers and may lack digital literacy skills.
Disability-Based: Websites designed without accessibility exclude people with visual, hearing, mobility, or cognitive disabilities.
Gender: In many parts of the world, women and girls have less access to technology and face cultural barriers.

üí° Info: According to the United Nations, about 2.7 billion people worldwide lack Internet access‚Äîroughly 37% of the global population. The divide is especially pronounced between developed and developing nations.

Web Accessibility: Making the Internet for Everyone

Web accessibility means designing websites so people with disabilities can use them effectively. This includes people who are:

  • Blind or low vision: Use screen readers and magnification tools
  • Deaf or hard of hearing: Need captions and transcripts for video and audio
  • Mobility impaired: May use voice control or keyboard-only navigation
  • Cognitively disabled: Need clear, simple language and consistent design

Web Content Accessibility Guidelines (WCAG)

The WCAG provides standards for making web content accessible. Key principles:

Perceivable: Content must be presented in ways users can perceive. Text alternatives for images allow blind users to understand visual content. Captions for videos help deaf users.
Operable: All functionality must be usable with keyboard navigation, not just mouse clicks. This helps mobility-impaired users.
Understandable: Information must be clear and simple. Plain language and consistent design help users with cognitive disabilities.
Robust: Websites must work with assistive technologies like screen readers. Proper HTML structure ensures compatibility.

üîç Example 6.6.1: Accessibility Features in Action
A blind user: Uses a screen reader that reads webpage text aloud. Images must have alt text describing them.
A deaf user: Watches a video with captions and reads a transcript.
A mobility-impaired user: Navigates using only keyboard. They can access all functionality through Tab key.
A user with dyslexia: Uses specialized reading tools and prefers sans-serif fonts with good spacing.
üí° Hint: When creating accessible websites, always include:
  • Alt text for all images
  • Captions/transcripts for video and audio
  • Keyboard navigation support
  • Clear, simple language
  • High contrast colors for readability
  • Proper heading structure (H1, H2, H3...)

Key Concepts & Terms

Digital Divide: The gap in Internet and technology access between different populations.
Web Accessibility: Designing digital content usable by people with disabilities.
Assistive Technology: Tools like screen readers, magnifiers, and voice recognition that help people with disabilities use computers.
Alt Text: Descriptive text for images that screen readers read aloud.
WCAG: International standards for web accessibility.

Now that you've created your website, let's ensure it's accessible to everyone, including people with disabilities.

Step 1: Add Alt Text to All Images

For every image on your site (logo, banner image, AI-generated image), add descriptive alt text. Click on the image, find the alt text field, and write a clear description of what the image shows. This helps blind users who use screen readers.

Step 2: Check Text Readability

Make sure all text is large enough (at least 14pt for body text). Use a sans-serif font like Arial or Roboto. Ensure good contrast between text color and background (dark text on light background). This helps users with low vision.

Step 3: Verify Video Accessibility

Check if your embedded video has captions. If not, look for a transcript. Add a text description below the video explaining what it shows. This helps deaf and hard of hearing users.

Step 4: Test Keyboard Navigation

Try navigating your site using only the Tab key and Enter key (no mouse). Can you click all buttons, follow all links, and access all content? This helps users with mobility disabilities.

Step 5: Use Clear, Simple Language

Review your text. Use simple, direct language. Break up long paragraphs. Use headings to organize content. Avoid jargon or explain technical terms. This helps users with cognitive disabilities and non-native speakers.

Step 6: Take a Final Screenshot

Capture a screenshot of your finished, accessible website. Submit this along with your other deliverables.

Step 7: Reflection

By making your site accessible, you've not only included people with disabilities‚Äîyou've made it better for everyone. Clear language helps all users. Good contrast helps anyone viewing in bright sunlight. Keyboard navigation helps power users. Accessibility is good design.

üìã Chapter Summary: The Internet

  • The Internet is a network of networks connecting billions of devices globally with no single owner
  • Search engines use crawling, indexing, and ranking to find and organize information, with results personalized to individual users

  • HTTPS adds encryption to protect sensitive data traveling between your browser and web servers
