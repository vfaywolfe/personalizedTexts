You've learned what functions are and why they're essential building blocks of programming. You understand that functions help organize code, avoid repetition, and create powerful abstractions. Imagine you're writing a recipe. A basic recipe might say "make a cake," but that's not very useful because it doesn't tell you what kind of cake or what size. üìö Key Concepts and Terms

Function Definition

The code that creates a function, specifying its name, parameters, and the instructions it will execute. In JavaScript, this begins with the function keyword.

Function Call (Function Invocation)

Executing a function by writing its name followed by parentheses. This runs the code inside the function.

Function Syntax

The specific structure and rules for writing function definitions in JavaScript: function name(parameters) { code }

Parameter

A variable listed in a function's definition that acts as a placeholder for values that will be passed in when the function is called.

Argument

The actual value passed to a function when calling it. Arguments fill in the parameters defined in the function.

Function Body

The code between the curly braces { } in a function definition. This is what executes when the function is called.

Curly Braces

The symbols { } that enclose the function body, marking the beginning and end of the function's code.

Defining Your First Function

Creating a function in JavaScript follows a specific pattern. Let's learn the exact syntax you need:

function functionName() { // Code to execute goes here }

Let's break down each component:

  • function ‚Äî The JavaScript keyword that tells the computer you're defining a function
  • functionName ‚Äî The name you choose for your function (follow the same rules as variable names)
  • () ‚Äî Parentheses that will eventually hold parameters (empty for now)
  • { } ‚Äî Curly braces that contain the function's code (the "function body")
Function names should use camelCase: start with a lowercase letter, then capitalize the first letter of each new word. Examples: drawHouse, calculateTotal, sayHello
Example 4.3.1: Creating a Simple Function
Step 1: Define the function

Let's create a function that displays a greeting message:

function sayHello() { fill(0); textSize(24); text("Hello, World!", 100, 200); }

This function is named sayHello. When called, it will set the fill color to black, set the text size to 24, and display "Hello, World!" at position (100, 200).

Step 2: Understand what we've created

Right now, we've only defined the function‚Äîwe've told JavaScript what sayHello means. The code inside won't run until we call the function.

Calling Functions

Defining a function creates it, but doesn't execute it. To run the code inside a function, you must call (or invoke) it:

sayHello();

When JavaScript sees this line, it looks for a function named sayHello, then executes all the code inside that function's curly braces.

Example 4.3.2: Complete Program with Function Definition and Call
// Function definition function sayHello() { fill(0); textSize(24); text("Hello, World!", 100, 200); } // Function call sayHello();

What happens:

  1. JavaScript reads the function definition and remembers what sayHello means
  2. JavaScript encounters sayHello(); and executes the function
  3. The text "Hello, World!" appears on the canvas
Forgetting to call the function is one of the most common mistakes beginners make! If you define a function but never call it, nothing will happen. The definition is like writing a recipe‚Äîcalling it is actually cooking the meal.
Example 4.3.3: Calling a Function Multiple Times

Once you've defined a function, you can call it as many times as you want:

function drawStar() { fill(255, 255, 0); beginShape(); vertex(0, -20); vertex(5, -5); vertex(20, 0); vertex(5, 5); vertex(0, 20); vertex(-5, 5); vertex(-20, 0); vertex(-5, -5); endShape(CLOSE); } translate(100, 100); drawStar(); translate(100, 0); drawStar(); translate(100, 0); drawStar();

This draws three identical stars at different positions. Notice how we only defined the star-drawing code once, but we can use it multiple times!

Adding Parameters to Functions

The functions we've created so far do exactly the same thing every time we call them. While this is useful, functions become much more powerful when they can accept parameters‚Äîinput values that customize their behavior.

Parameters are variables you list inside the parentheses when defining a function. They act as placeholders for actual values (called arguments) that will be provided when the function is called.

Parameters vs. Arguments ‚Äî Important Distinction!
Parameters are the placeholder variables in the function definition.
Arguments are the actual values you pass when calling the function.
Think of parameters as empty boxes and arguments as what you put in those boxes.
Example 4.3.4: Function with One Parameter
Step 1: Define a function with a parameter
function greetPerson(name) { fill(0); textSize(20); text("Hello, " + name + "!", 100, 200); }

The word name in the parentheses is a parameter. Inside the function, name acts like a variable that will hold whatever value we pass in.

Step 2: Call the function with different arguments
greetPerson("Alice"); // Displays: "Hello, Alice!" greetPerson("Bob"); // Displays: "Hello, Bob!" greetPerson("Charlie"); // Displays: "Hello, Charlie!"

Each time we call the function, we pass a different argument (the actual value). The argument fills in the name parameter.

Step 3: How it works

When you call greetPerson("Alice"):

  1. JavaScript assigns "Alice" to the parameter name
  2. Inside the function, wherever name appears, it has the value "Alice"
  3. The function displays "Hello, Alice!"
Parameters make your functions flexible! Instead of writing separate functions for greeting Alice, Bob, and Charlie, you write one function that can greet anyone by accepting their name as a parameter.
Example 4.3.5: Function with Multiple Parameters

Functions can accept multiple parameters, separated by commas:

function drawCustomCircle(x, y, diameter) { ellipse(x, y, diameter, diameter); } // Now we can draw circles anywhere with any size! drawCustomCircle(100, 100, 50); // Small circle at (100, 100) drawCustomCircle(300, 200, 150); // Large circle at (300, 200) drawCustomCircle(200, 300, 75); // Medium circle at (200, 300)

How the parameters work:

  • When we call drawCustomCircle(100, 100, 50)
  • The first argument (100) fills the first parameter (x)
  • The second argument (100) fills the second parameter (y)
  • The third argument (50) fills the third parameter (diameter)
  • Arguments are matched to parameters by position
When calling a function with parameters, you must provide the correct number of arguments in the correct order! If the function expects 3 parameters and you only provide 2 arguments, the missing parameter will be undefined and your function may not work correctly.
Example 4.3.6: Building a Flexible Drawing Function

Let's create a function that draws a house at any position with any size:

function drawHouse(x, y, size) { // Draw walls fill(165, 42, 42); rect(x, y, size, size); // Draw roof fill(139, 69, 19); triangle(x, y, x + size/2, y - size/2, x + size, y); // Draw door fill(139, 69, 19); var doorWidth = size * 0.3; var doorHeight = size * 0.5; rect(x + size/2 - doorWidth/2, y + size - doorHeight, doorWidth, doorHeight); // Draw window fill(173, 216, 230); var windowSize = size * 0.2; rect(x + size * 0.2, y + size * 0.2, windowSize, windowSize); } // Draw houses of different sizes at different positions background(135, 206, 235); drawHouse(50, 200, 100); // Small house drawHouse(200, 150, 150); // Medium house drawHouse(400, 180, 120); // Another house

This function is incredibly powerful because everything is based on the parameters. The same function can draw tiny houses, huge houses, or houses anywhere on the canvas!

Naming Parameters: Choose descriptive parameter names that indicate what values they represent. Use x and y for coordinates, size or width and height for dimensions, color for colors, etc. Good parameter names make your functions self-documenting!

Term Definition

Question 3 (Code Analysis):

Given this function: function drawSquare(x, y, size)
Which function call is correct?

Show Answer

Answer: C) drawSquare(100, 200, 50);

The function expects three arguments (x, y, and size). Option C correctly provides all three. Option A has no arguments, option B is missing the third argument, and option D incorrectly tries to use the parameter names instead of actual values.

Question 4 (Code Writing):

Write a function called drawRectangle that accepts four parameters: x, y, width, and height. The function should draw a rectangle using these parameters.

Show Sample Answer
function drawRectangle(x, y, width, height) { rect(x, y, width, height); }

This function accepts four parameters and uses them to draw a rectangle. You could call it with: drawRectangle(50, 50, 100, 75);

Question 5 (Application):

Write a complete program that defines a function called displayMessage with two parameters: message and size. The function should display the message at position (100, 200) with the specified text size. Then call the function twice with different messages and sizes.

Show Sample Answer
// Function definition function displayMessage(message, size) { fill(0); textSize(size); text(message, 100, 200); } // Function calls displayMessage("Hello!", 24); displayMessage("Welcome to programming!", 16);

Note: The second message will overwrite the first since they're at the same position. In a real program, you might want to use different y positions!
