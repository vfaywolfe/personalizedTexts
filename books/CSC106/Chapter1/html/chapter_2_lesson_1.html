Key Concepts and Terms

Canvas

A rectangular drawing surface in computer graphics where shapes, text, and images are rendered. In this course, we'll work with a 400x400 pixel canvas.

Coordinate System

A system for identifying locations on the canvas using x (horizontal) and y (vertical) values. The origin point (0, 0) is located at the top-left corner of the canvas.

Pixel

The smallest unit of measurement on a digital display. Our canvas is measured in pixels, with dimensions of 400 pixels wide by 400 pixels tall.

Function

A named block of code that performs a specific task. Functions can accept parameters (inputs) to customize their behavior. Examples include ellipse(), rect(), and line().

Parameters

Values passed into a function to control its behavior. For example, in ellipse(200, 200, 100, 100), the parameters specify the center position and size of the ellipse.

RGB Color Model

A color representation system using three values: Red, Green, and Blue. Each value ranges from 0 to 255, where (0, 0, 0) is black and (255, 255, 255) is white.

Comment

Text in code that is ignored by the computer but helps humans understand what the code does. In JavaScript, comments begin with //.

Stroke

The outline or border of a shape. Can be customized with color using stroke() and thickness using strokeWeight().

Fill

The interior color of a shape, set using the fill() function with RGB values.

Rendering Order

The sequence in which drawing commands are executed. Shapes drawn later appear on top of shapes drawn earlier, similar to layering paper cutouts.

Before we start drawing, it's crucial to understand how the coordinate system works in computer graphics. Unlike the mathematical coordinate system you may have learned in school, computer graphics use a slightly different approach.

The Origin Point

In computer graphics, the origin point (0, 0) is located at the top-left corner of the canvas, not the center. This is an important distinction!

The coordinate system in computer graphics places (0, 0) at the top-left corner because early computer displays drew pixels starting from the top-left, moving left-to-right and top-to-bottom‚Äîsimilar to how we read English text.

Understanding X and Y Values

Axis Direction Range (400x400 canvas)
X-axis Horizontal (left to right) 0 (left edge) to 400 (right edge)
Y-axis Vertical (top to bottom) 0 (top edge) to 400 (bottom edge)
To find the center of a 400x400 canvas, divide both dimensions by 2. The center point is at coordinates (200, 200).

Key Coordinate Examples

Example 2.1.1: Understanding Key Canvas Positions

Let's identify some important positions on our 400x400 canvas:

  • (0, 0) - Top-left corner
  • (400, 0) - Top-right corner
  • (0, 400) - Bottom-left corner
  • (400, 400) - Bottom-right corner
  • (200, 200) - Center of canvas
  • (100, 50) - 100 pixels from left, 50 pixels from top
Values outside the canvas range (negative numbers or numbers greater than 400) will position shapes partially or completely off-screen!

Drawing with the ellipse() Function

The ellipse() function is your first tool for creating visual elements. It can draw both perfect circles and stretched ovals.

Ellipse Function Syntax

ellipse(x, y, width, height);

Parameters Explained

  • x - The horizontal position of the ellipse's center
  • y - The vertical position of the ellipse's center
  • width - The horizontal diameter of the ellipse
  • height - The vertical diameter of the ellipse
Notice that the x and y values position the center of the ellipse, not its top-left corner. This makes it easy to position circles exactly where you want them!
Example 2.1.2: Drawing a Perfect Circle
Step 1: To draw a perfect circle, make the width and height equal.
ellipse(200, 200, 150, 150);
Step 2: Understand what this code does:
  • x = 200: positions the center 200 pixels from the left edge
  • y = 200: positions the center 200 pixels from the top edge
  • width = 150: makes the circle 150 pixels wide
  • height = 150: makes the circle 150 pixels tall
Result: A perfect circle with a 150-pixel diameter, centered in the canvas.
Example 2.1.3: Drawing an Oval
Step 1: To draw an oval, use different values for width and height.
ellipse(200, 200, 250, 130);
Step 2: This creates a horizontally-stretched oval:
  • width = 250: wider than it is tall
  • height = 130: creates the vertical dimension
Result: An oval shape that's wider than it is tall, perfect for a smile!
When equal width and height values create a circle, what shape results when width is much larger than height? Try it in your code!

Building a Complete Drawing: The Smiley Face

Now let's combine multiple ellipses to create a complete composition‚Äîa simple smiley face. This will demonstrate how individual shapes work together to create recognizable images.

Example 2.1.4: Complete Smiley Face
Step 1: Draw the face (a large circle).
ellipse(200, 200, 300, 300); // Main face
Step 2: Add the left eye.
ellipse(150, 150, 40, 40); // Left eye
Step 3: Add the right eye (notice the symmetry).
ellipse(250, 150, 40, 40); // Right eye
Step 4: Create the smile using a wide, short oval.
ellipse(200, 240, 120, 80); // Smile
Complete Code:
// Draw a smiley face ellipse(200, 200, 300, 300); // Main face ellipse(150, 150, 40, 40); // Left eye ellipse(250, 150, 40, 40); // Right eye ellipse(200, 240, 120, 80); // Smile
Notice how comments (the text after //) help explain what each line does. This makes your code much easier to understand when you come back to it later!

Organizing Code with Comments

As your programs grow more complex, comments become essential for understanding and maintaining your code. Comments are notes to yourself and other programmers‚Äîthe computer completely ignores them.

Comment Syntax

Good comments explain why you're doing something or what a complex piece of code does, not just repeating what's obvious from the code itself.

Comments are also useful for temporarily disabling code without deleting it‚Äîa technique called "commenting out."

The rect() function lets you draw rectangles and squares‚Äîessential shapes for many designs!

Rectangle Function Syntax

rect(x, y, width, height);
Critical difference: Unlike ellipse(), the x and y parameters for rect() specify the top-left corner of the rectangle, not the center!

Parameters Explained

  • x - The horizontal position of the rectangle's top-left corner
  • y - The vertical position of the rectangle's top-left corner
  • width - How wide the rectangle is
  • height - How tall the rectangle is
Example 2.1.6: Drawing a Square
Step 1: To draw a square, make width equal to height.
rect(150, 150, 100, 100);
Step 2: Understanding the positioning:
  • x = 150: The left edge starts 150 pixels from the left
  • y = 150: The top edge starts 150 pixels from the top
  • The right edge will be at 150 + 100 = 250
  • The bottom edge will be at 150 + 100 = 250
Result: A 100x100 pixel square with its top-left corner at (150, 150).
Example 2.1.7: Robot Face Using Rectangles

Let's recreate our smiley face using rectangles to make it look more robotic:

// Robot face using rectangles rect(100, 100, 200, 200); // Head (square) rect(140, 150, 30, 30); // Left eye rect(230, 150, 30, 30); // Right eye rect(150, 220, 100, 20); // Mouth
To center a rectangle on the canvas, subtract half its width from 200 for the x-coordinate, and half its height from 200 for the y-coordinate.

Drawing Lines with line()

The line() function draws a straight line between two points. Unlike shapes, lines don't have a fill‚Äîonly a stroke.

Line Function Syntax

line(x1, y1, x2, y2);

Parameters Explained

  • x1 - The x-coordinate of the starting point
  • y1 - The y-coordinate of the starting point
  • x2 - The x-coordinate of the ending point
  • y2 - The y-coordinate of the ending point
Example 2.1.8: Drawing a Horizontal Line
Step 1: To draw a horizontal line, keep y1 and y2 the same.
line(100, 200, 300, 200);
Result: A horizontal line from point (100, 200) to point (300, 200), running across the middle of the canvas.
Example 2.1.9: Drawing a Diagonal Line
Step 1: To draw a diagonal, change both x and y coordinates.
line(100, 100, 300, 300);
Result: A diagonal line from the upper-left region (100, 100) to the lower-right region (300, 300).
Example 2.1.10: Adding Eyebrows to Robot Face
// Robot face with eyebrows rect(100, 100, 200, 200); // Head rect(140, 150, 30, 30); // Left eye rect(230, 150, 30, 30); // Right eye rect(150, 220, 100, 20); // Mouth // Add eyebrows with lines line(140, 140, 170, 140); // Left eyebrow line(230, 140, 260, 140); // Right eyebrow

Understanding Drawing Order (Layering)

One of the most important concepts in graphics programming is understanding how the order of your code affects what you see on screen. Shapes drawn later will appear on top of shapes drawn earlier.

Think of it like layering paper cutouts: each new shape is placed on top of the previous ones. This is called the "z-order" or "painting order."
Example 2.1.11: Demonstrating Drawing Order

Code Example 1: Red circle drawn first

// Red circle first, then blue circle fill(255, 0, 0); // Red ellipse(180, 200, 150, 150); fill(0, 0, 255); // Blue ellipse(220, 200, 150, 150);

Result: The blue circle appears on top of the red circle.

Code Example 2: Blue circle drawn first

// Blue circle first, then red circle fill(0, 0, 255); // Blue ellipse(220, 200, 150, 150); fill(255, 0, 0); // Red ellipse(180, 200, 150, 150);

Result: The red circle appears on top of the blue circle.

Be careful with drawing order! If you draw a background shape last, it will cover up everything you drew before it.

Color brings your drawings to life! In computer graphics, we create colors by mixing three primary colors of light: Red, Green, and Blue (RGB).

Understanding RGB Color Values

Each color component (Red, Green, Blue) uses a value from 0 to 255:

  • 0 = none of that color
  • 255 = maximum amount of that color
  • Values in between = partial amounts
Color Red Green Blue RGB Code
Black 0 0 0 (0, 0, 0)
White 255 255 255 (255, 255, 255)
Red 255 0 0 (255, 0, 0)
Green 0 255 0 (0, 255, 0)
Blue 0 0 255 (0, 0, 255)
Yellow 255 255 0 (255, 255, 0)
Cyan 0 255 255 (0, 255, 255)
Magenta 255 0 255 (255, 0, 255)
Use an RGB color picker tool (search "RGB color picker" online) to experiment with creating custom colors!

Setting the Background Color

The background() function sets the canvas background color.

background(red, green, blue); // Examples: background(135, 206, 235); // Sky blue background(255, 255, 255); // White background(0, 0, 0); // Black
The background() function is typically called first in your program, before drawing any shapes.

Setting Fill Colors

The fill() function sets the interior color of shapes.

fill(red, green, blue); // Example: Draw a red circle fill(255, 0, 0); // Set fill to red ellipse(200, 200, 100, 100);
The fill() function affects ALL shapes drawn after it, until you call fill() again with a different color!
Example 2.1.12: Traffic Light

Let's create a simple traffic light with three colored circles:

// Gray background for the light housing background(80, 80, 80); // Red light (top) fill(255, 0, 0); ellipse(200, 100, 60, 60); // Yellow light (middle) fill(255, 255, 0); ellipse(200, 200, 60, 60); // Green light (bottom) fill(0, 255, 0); ellipse(200, 300, 60, 60);

Controlling Strokes (Outlines)

You can customize the outline of shapes using three functions:

stroke(red, green, blue); // Sets the stroke color strokeWeight(thickness); // Sets the stroke thickness in pixels noStroke(); // Removes the stroke entirely
Example 2.1.13: Customizing Strokes
// Circle with thick red outline stroke(255, 0, 0); // Red stroke strokeWeight(8); // 8 pixels thick fill(255, 255, 0); // Yellow fill ellipse(150, 200, 100, 100); // Circle with no outline noStroke(); // Remove stroke fill(0, 255, 0); // Green fill ellipse(250, 200, 100, 100);
Like fill(), stroke settings persist until changed. Use noStroke() when you want shapes without outlines, like in the sunny snowy day example!

Building a Complete Scene

Now let's put everything together to create a complete, colorful scene. We'll build a snowman on a sunny day!

Example 2.1.14: Sunny Snowy Day (Complete Scene)
Step 1: Set up the background (sky)
background(135, 206, 235); // Sky blue background
Step 2: Draw the ground
noStroke(); // No outline needed fill(255, 255, 255); // White snow rect(0, 300, 400, 100); // Ground extending across canvas
Step 3: Draw the sun
fill(255, 255, 0); // Bright yellow ellipse(80, 64, 100, 100); // Sun in upper left
Step 4: Build the snowman (bottom to top)
// Snowman body (white circles, no stroke) fill(255, 255, 255); ellipse(200, 350, 150, 150); // Bottom ellipse(200, 250, 100, 100); // Middle ellipse(200, 170, 75, 75); // Head
Step 5: Add snowman features
// Eyes (black circles) fill(0, 0, 0); ellipse(190, 165, 8, 8); // Left eye ellipse(210, 165, 8, 8); // Right eye // Nose (orange triangle - using a thin rectangle for simplicity) fill(255, 140, 0); // Orange rect(198, 175, 4, 15); // Buttons (black circles) fill(0, 0, 0); ellipse(200, 240, 10, 10); ellipse(200, 260, 10, 10); ellipse(200, 280, 10, 10);
