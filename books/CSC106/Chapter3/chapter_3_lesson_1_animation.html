üìñ Key Concepts and Terms

Animation

The illusion of motion created by displaying a sequence of images (frames) in rapid succession. Each image differs slightly from the previous one, and when shown quickly enough, our eyes perceive smooth, continuous movement instead of individual still pictures.

Frame

A single image in an animation sequence. Computer animations typically display 30-60 frames per second (fps) to create smooth motion. Each frame shows the scene at one specific moment in time.

Flipbook Animation

A traditional animation technique where sequential images are drawn on separate pages. When you flip through the pages quickly, the images blend together to create the illusion of movement. This is the foundational concept behind all animation.

draw() Function

A special function that executes repeatedly and continuously in a loop, typically 60 times per second. This is the engine that powers animation in programming‚Äîeach time draw() runs, it creates one frame of your animation.

Increment Operator (+=)

A shorthand operator that adds a value to a variable and stores the result back in that same variable. For example, x += 5 is equivalent to x = x + 5. This is commonly used in animations to move objects.

Decrement Operator (-=)

A shorthand operator that subtracts a value from a variable. For example, y -= 3 is equivalent to y = y - 3. Useful for moving objects in the opposite direction or shrinking sizes.

Multiply-Assign Operator (*=)

A shorthand operator that multiplies a variable by a value. For example, size *= 1.1 is equivalent to size = size * 1.1, which increases the size by 10% each time.

Increment by One (++)

The most concise way to add 1 to a variable. count++ is equivalent to count += 1 or count = count + 1. Often used for counters and simple animations.

üé® Understanding Animation

What Is Animation Really?

Before jumping into coding animations, let's first understand what an animation really is. How would you create an animation without any programming at all?

Imagine drawing a series of pictures on separate pieces of paper‚Äîeach picture slightly different from the one before it. If you flip through those pages quickly, your eyes blend the images together and you see motion instead of just a series of still pictures. This technique is called flipbook animation, and it's the foundation of all animations‚Äîfrom classic Disney movies to modern video games.

Example 3.1.1: Flipbook Animation Concept

Imagine a pencil-and-paper animation showing a car driving down the street:

  • Frame 1: Car at position x = 0
  • Frame 2: Car at position x = 10
  • Frame 3: Car at position x = 20
  • Frame 4: Car at position x = 30
  • ... and so on

When flipped fast enough, it looks like the car is smoothly driving across the page, even though it's just a collection of individual drawings.

This simple concept helps explain how animations work in programming: by continuously redrawing images with small changes over time, the computer creates the illusion of movement. Once you understand this, you'll be ready to write your own animated programs!

From Paper to Pixels

Creating a flipbook animation probably takes quite a bit of time and effort‚Äîand honestly, it's not very smooth. The motion can be shaky, and there's no color or detail to make it visually appealing.

But we live in the digital age! With programming, you can create much smoother, more colorful, and complex animations in just a few minutes, using only a dozen lines of code or so.

Computer-generated animations look cleaner and more fluid compared to hand-drawn flipbooks‚Äîand require far less manual work! Instead of drawing hundreds of frames by hand, you write code once and let the computer generate all the frames automatically.

‚öôÔ∏è The draw() Function: Your Animation Engine

Now that you understand the concept of animation, let's learn how to create it with code. The secret to animation in programming is a special function called draw().

How draw() Works

Unlike regular code that runs once from top to bottom and then stops, the draw() function runs continuously in a loop‚Äîtypically 60 times per second! Each time draw() executes, it creates one frame of your animation.

Example 3.1.2: Basic Animation Structure

Here's the basic structure of an animated program:

// Variables defined outside draw() var x = 10; // This function runs repeatedly draw = function() { background(255, 255, 255); // Clear the canvas ellipse(x, 200, 50, 50); // Draw a circle x = x + 2; // Move the circle };

What happens:

  1. The variable x starts at 10
  2. The draw() function runs automatically
  3. It draws a white background, then draws a circle at position (x, 200)
  4. It increases x by 2
  5. The function runs again, now with x = 12
  6. This repeats 60 times per second, making the circle move smoothly to the right
Variables used in animation must be declared outside the draw() function. If you declare them inside draw(), they'll reset to their starting value every frame, and your animation won't work!

The Importance of background()

Notice that we call background() at the beginning of the draw() function. This is crucial for creating clean animations!

Example 3.1.3: With and Without background()
// WITHOUT background() - leaves trails var x = 10; draw = function() { // No background call here! ellipse(x, 200, 50, 50); x = x + 2; }; // WITH background() - clean animation var x = 10; draw = function() { background(255, 255, 255); // Clears previous frames ellipse(x, 200, 50, 50); x = x + 2; };

Without background(): Each circle stays on the canvas, creating a trail effect. While this might be intentional for some designs, it's usually not what you want.

With background(): Each frame starts fresh with a clean slate, making the circle appear to move smoothly without leaving traces behind.

Think of background() as an eraser that wipes the canvas clean before drawing the next frame. Without it, every frame gets drawn on top of all previous frames!

üöÄ Shortcut Operators for Efficient Animation

You've seen that we can update variables to create animation using code like x = x + 2. While this works perfectly, programmers love shortcuts! JavaScript provides several operators that make updating variables more concise and easier to read.

The += Operator (Add and Assign)

Instead of writing x = x + 2, you can write x += 2. Both do exactly the same thing, but += is shorter and clearer about your intent.

Example 3.1.4: Using the += Operator
// Traditional way x = x + 5; // Shortcut way (identical result) x += 5; // In an animation var x = 200; var y = 150; draw = function() { background(155, 200, 71); fill(191, 0, 255); ellipse(x, y, 100, 100); // Move diagonally using += shortcuts x += 1; // Same as: x = x + 1 y += 2; // Same as: y = y + 2 };

What this does: The purple ball moves diagonally across the screen (1 pixel right and 2 pixels down each frame), creating smooth diagonal motion.

The -= Operator (Subtract and Assign)

Similarly, x -= 3 is a shortcut for x = x - 3. This is useful for moving objects in the opposite direction or shrinking sizes.

Example 3.1.5: Moving in Different Directions
var x = 200; var y = 350; draw = function() { background(155, 200, 71); fill(191, 0, 255); ellipse(x, y, 100, 100); x += 1; // Move right (increase x) y -= 2; // Move up (decrease y) };

Result: The ball moves diagonally up and to the right (right 1 pixel, up 2 pixels each frame).

The *= Operator (Multiply and Assign)

The *= operator multiplies a variable by a value. This is perfect for exponential growth effects like expanding circles or accelerating objects.

Example 3.1.6: Shrinking Ball Animation
var ballWidth = 300; var ballHeight = 200; draw = function() { background(155, 200, 71); fill(191, 0, 255); ellipse(200, 200, ballWidth, ballHeight); // Shrink by 1% each frame ballWidth *= 0.99; // Same as: ballWidth = ballWidth * 0.99 ballHeight *= 0.99; // Same as: ballHeight = ballHeight * 0.99 };

Result: The ball gradually shrinks. Each frame, it becomes 99% of its previous size (or shrinks by 1%).

Multiplying by values less than 1 (like 0.99) makes things shrink. Multiplying by values greater than 1 (like 1.05) makes things grow!

The ++ and -- Operators (Increment and Decrement by 1)

The most common animation update is adding or subtracting 1. For this, there's an even shorter shortcut:

Example 3.1.7: Three Ways to Add 1
// These three lines all do the exact same thing: x = x + 1; // Traditional way x += 1; // Shortcut #1 x++; // Shortcut #2 (most concise) // Similarly, these three lines all subtract 1: y = y - 1; // Traditional way y -= 1; // Shortcut #1 y--; // Shortcut #2 (most concise)

Which should you use? All three work identically! Many programmers prefer += because it's versatile (you can easily change the increment amount from 1 to any other number), but ++ is perfectly valid and very common in professional code.

Choose the style that makes sense to you. If you're incrementing by 1, you can use ++. If you're incrementing by any other amount, += is required. Both are good choices!

üé™ Creating Complex Animations

Now that you understand the draw() function and shortcut operators, let's combine them to create more sophisticated animations. The real power of animation comes from changing multiple properties at once!

Example 3.1.8: Multi-Property Animation - Diagonal Shrinking Ball

Let's create a ball that moves diagonally while gradually shrinking:

noStroke(); var x = 200; var y = 350; var ballWidth = 300; var ballHeight = 200; draw = function() { background(155, 200, 71); fill(191, 0, 255); ellipse(x, y, ballWidth, ballHeight); // Update multiple properties each frame x += 1; // Move right y -= 2; // Move up ballWidth *= 0.99; // Shrink width by 1% ballHeight *= 0.99; // Shrink height by 1% };

What happens: Each frame, four things occur simultaneously:

  1. The ball moves 1 pixel to the right (x increases)
  2. The ball moves 2 pixels up (y decreases)
  3. The ball's width shrinks to 99% of its previous size
  4. The ball's height shrinks to 99% of its previous size

The result: A purple ball that floats diagonally upward while gracefully shrinking until it disappears!

You can animate any property that uses numbers: position (x, y), size (width, height), colors (red, green, blue values), rotation angles, transparency‚Äîanything! Just update the variable values inside the draw() function.

Practical Example: Animated Sunrise

Example 3.1.9: Sunrise Animation

Let's create a sunrise where the sun rises from the horizon while growing larger:

noStroke(); // Starting size for the sun var sunSize = 30; draw = function() { // Beautiful blue sky background(82, 222, 240); // The sun - starts small near the horizon fill(255, 204, 0); ellipse(200, 290, sunSize, sunSize); // The land, blocking half of the sun fill(76, 168, 67); rect(0, 300, 400, 100); // Make the sun grow larger sunSize += 1; };

Enhancement idea: You could add a second variable for the sun's y-position and make it rise while growing!

‚ö†Ô∏è Common Animation Pitfalls and Solutions

Pitfall #1: Variables Declared Inside draw()

If you declare variables inside draw(), they reset every frame and your animation won't work. Always declare animation variables outside draw()!

Pitfall #2: Forgetting background()

Without background() at the start of draw(), objects leave trails. While sometimes intentional, this usually creates messy, unreadable animations.

Pitfall #3: Objects Disappearing Off Screen

If objects move off the edge of the canvas, they disappear! In later lessons, you'll learn how to detect edges and make objects bounce back or wrap around.

Pitfall #4: Too Fast or Too Slow

If your animation moves too fast, use smaller increment values (like x += 0.5 instead of x += 5). If it's too slow, use larger values. Experiment to find the right speed!
