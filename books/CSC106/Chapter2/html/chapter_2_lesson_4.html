üìö Concepts and Terms

Variable
A named storage location in computer memory that holds a value. Variables allow programmers to store data that can be used and modified throughout a program. Think of a variable as a labeled box where you can put information and retrieve it later by referencing its name.
Declaration
The process of creating a new variable using the var keyword. Declaration tells the computer to reserve a space in memory for storing a value. Example: var x;
Initialization
Assigning an initial value to a variable when it is declared. This combines declaration and assignment in one step. Example: var x = 5;
Assignment
The process of giving a value to a variable using the equals sign (=). The value on the right side is stored in the variable on the left side. Example: x = 10;
Reassignment
Changing the value stored in an existing variable. This replaces the old value with a new one. Example: if x was 5, writing x = 9; changes it to 9.
Expression
A combination of values, variables, and operators that the computer evaluates to produce a result. Examples: x + 5, width * 2, total + 1
Identifier
The name given to a variable. Good identifiers are descriptive and follow naming conventions (start with a letter, use camelCase for multiple words, no spaces). Examples: circleSize, playerScore, x
Memory Diagram
A visual tool used to trace program execution by showing variable names (identifiers) and their current values. Memory diagrams help programmers understand how variables change as code runs line by line.
Data Type
The kind of value a variable can store. In JavaScript, common data types include numbers (like 42 or 3.14), strings (text like "hello"), and booleans (true or false). JavaScript is dynamically typed, meaning variables can hold any type of value.

üíª Understanding Variables

What is a Variable?

A variable is a fundamental building block of programming‚Äîit's a named container that stores a value in your computer's memory. Just as you might use a labeled box to store items in your home, variables let you store and organize data in your programs.

The power of variables comes from their flexibility. Instead of writing the number 50 everywhere you need it, you can store 50 in a variable called circleSize and use that variable name instead. Then, if you want all your circles to be larger, you only need to change one number‚Äîthe value stored in circleSize‚Äîand every circle that uses that variable will automatically update.

Variables transform programming from rigid, repetitive tasks into flexible, powerful problem-solving. Here's why they're essential:

  • Maintainability: Change one value instead of hunting through hundreds of lines of code

üîß Creating and Using Variables

Variable Declaration and Initialization

In JavaScript, we create variables using the var keyword. There are two ways to create a variable:

Example 2.3.1: Declaration Only

Step 1: Declare a variable without assigning a value:

var x;

Explanation: This creates a variable named x but doesn't give it a value yet. The variable exists in memory but contains undefined until we assign it a value.

When to use: Rarely! Usually, you'll want to initialize variables with values right away.

Example 2.3.2: Declaration with Initialization

Step 1: Declare a variable and assign it a value in one statement:

var circleSize = 100;

Explanation:

  • var - The keyword that tells JavaScript we're creating a new variable
  • circleSize - The identifier (name) we're giving to this variable
  • = - The assignment operator that stores a value in the variable
  • 100 - The initial value being stored

Result: A variable named circleSize now exists and contains the value 100.

The equals sign (=) in programming doesn't mean "equals" like it does in math! Instead, it means "assign the value on the right to the variable on the left." Think of it as an arrow pointing left: circleSize ‚Üê 100

Using Variables in Code

Once you've created a variable, you can use its name anywhere you would normally use a value:

Example 2.3.3: Using Variables in Drawing Functions

Without Variables:

ellipse(200, 200, 100, 100); // If we want to change size, we change 4 numbers ellipse(300, 200, 100, 100); // Same size, have to change again ellipse(250, 300, 100, 100); // And again...

With Variables:

var circleSize = 100; // Define size once ellipse(200, 200, circleSize, circleSize); // Use the variable ellipse(300, 200, circleSize, circleSize); // Reuse it ellipse(250, 300, circleSize, circleSize); // Much easier!

Benefit: Now if you want all three circles to be size 150 instead of 100, you only change ONE number (the value of circleSize), and all three circles automatically update!

When JavaScript sees a variable name in your code, it looks up that variable's current value and uses it. So ellipse(200, 200, circleSize, circleSize) is exactly the same as writing ellipse(200, 200, 100, 100) when circleSize is 100.

Variables in Expressions

Variables become even more powerful when used in mathematical expressions:

Example 2.3.4: Variables in Arithmetic Expressions
var x = 200; var y = 250; // Draw a face using variables ellipse(x, y, 200, 200); // Face centered at (200, 250) ellipse(x - 50, y - 50, 40, 40); // Left eye ellipse(x + 50, y - 50, 40, 40); // Right eye ellipse(x, y + 20, 120, 130); // Mouth

Explanation:

  • All positions are based on x and y
  • Eyes are positioned 50 pixels left and right of center
  • To move the entire face, just change x and y!

Power of This Approach: Change x from 200 to 300, and the entire face (all four shapes) moves 100 pixels to the right automatically!

üîÑ Changing Variable Values

Reassignment: Updating Variables

One of the most powerful features of variables is that you can change their values after they're created. This is called reassignment.

Example 2.3.5: Variable Reassignment

Step 1: Create a variable and use it:

var x = 5; println(x); // Prints: 5

Step 2: Change the variable's value (note: no var keyword!):

x = 9; // Reassignment - no "var" needed println(x); // Prints: 9

Key Point: When reassigning, you DON'T use var again. The var keyword is only for creating new variables. To change an existing variable, just use the variable name, equals sign, and new value.

Using var when reassigning a variable creates a NEW variable with the same name, which can cause confusing bugs! When changing a variable's value, never use var again‚Äîjust write variableName = newValue;

Variables and Expressions in Assignment

You can use a variable's current value to calculate its new value:

Example 2.3.6: Using Variables in Their Own Reassignment
var score = 10; score = score + 5; // Take current value (10), add 5, store result (15) back in score println(score); // Prints: 15 score = score * 2; // Take current value (15), multiply by 2, store result (30) println(score); // Prints: 30

How This Works:

  1. JavaScript first evaluates the RIGHT side of the equals sign
  2. It looks up the current value of score (10)
  3. It performs the calculation (10 + 5 = 15)
  4. It stores the result (15) back into score, replacing the old value
The pattern x = x + 1; is so common in programming that it has a name: "incrementing." It increases a variable's value by 1. You'll use this constantly when you learn about animation and loops later!

Practical Example: Scalable Drawings

Example 2.3.7: Creating a Scalable Face
// Define the face size var faceSize = 100; var mouthSize = 30; // Draw the face ellipse(200, 200, faceSize, faceSize); // Face ellipse(200, 220, mouthSize, mouthSize); // Mouth // To make the whole face bigger, just change faceSize! // Try changing faceSize to 150 and see what happens

Challenge: What if you want the mouth to always be 30% of the face size? Change the code to:

var faceSize = 100; var mouthSize = faceSize * 0.3; // Always 30% of face size ellipse(200, 200, faceSize, faceSize); ellipse(200, 220, mouthSize, mouthSize);

Now when you change faceSize, the mouth automatically scales proportionally!

üß† Memory Diagrams: Visualizing Variables

Memory diagrams are powerful tools that help you understand how variables work "under the hood." They show you exactly what's happening in your computer's memory as your program runs, line by line.

What are Memory Diagrams?

A memory diagram is a visual representation of your program's variables at a specific moment in time. They're typically hand-drawn with an "inverted T" shape that has two columns:

  • Identifier (name) column: The variable names
  • Value (stack) column: The values stored in those variables
Memory diagrams are especially useful when debugging code or when you're trying to understand why a variable doesn't have the value you expected. They're a fundamental tool that professional programmers use when reasoning about code!

How to Create a Memory Diagram

Follow these steps to trace code and create a memory diagram:

üìã Memory Diagram Rules:
  1. Read code from top to bottom, line by line
  2. When you see var, add a new row to your diagram from bottom to top
  3. When you see an assignment without var, cross out the old value and write the new one
  4. To find a variable's current value, read the diagram top to bottom and use the first non-crossed-out value

Worked Example: Tracing with Memory Diagrams

Example 2.3.8: Complete Memory Diagram Trace

Let's trace this code step by step:

/* Line 1 */ var x = 5; /* Line 2 */ var y = 7; /* Line 3 */ x = 9; /* Line 4 */ var total = x + y; /* Line 5 */ println(total);

Step 1: After Line 1 executes:

identifier value
x 5

We allocated memory for x and stored 5

Step 2: After Line 2 executes:

identifier value
y 7
x 5

We add new variables from bottom to top

Step 3: After Line 3 executes:

identifier value
y 7
x 5 9

We cross out the old value and write the new one

Step 4: After Line 4 executes:

identifier value
total 16
y 7
x 5 9

We calculate x + y = 9 + 7 = 16 and store in total

Step 5: Line 5 prints the value of total (which is 16) but doesn't change memory, so the diagram stays the same.

Final Answer: The program prints 16 to the console.

Common mistake: If Line 3 had been var x = 9; (with var), it would create a SECOND variable also named x! This creates confusion because now there are two x's in memory. The newer one (at the top) would be used. This wastes memory and is considered a bug. Always use var only when creating a NEW variable.

üìù Variable Naming Best Practices

Choosing good variable names is crucial for writing readable, maintainable code. Well-named variables make your code self-documenting‚Äîsomeone reading your code can understand what it does without extensive comments.

Naming Rules (Must Follow)

‚úÖ JavaScript Variable Naming Rules:
  • Must start with a letter, underscore (_), or dollar sign ($)
  • Cannot start with a number
  • Can contain letters, numbers, underscores, and dollar signs
  • Cannot use JavaScript reserved words (like var, if, for, etc.)
  • Are case-sensitive (myVar and myvar are different variables)
  • Cannot contain spaces or special characters (@, #, %, etc.)
‚ùå Invalid Variable Names:
var 2cool = 50; // Can't start with a number var my-variable = 10; // Can't use hyphens var user name = "Bob"; // Can't have spaces var var = 5; // Can't use reserved words

Naming Conventions (Should Follow)

Beyond the strict rules, follow these conventions to write professional code:

üíº Best Practices:
  • Use camelCase: Start with lowercase, capitalize each new word: circleSize, playerScore, eyeDistance
  • Be descriptive: Use names that clearly describe what the variable stores: studentAge instead of a
  • Avoid abbreviations: Write radius instead of rad (unless the abbreviation is universally understood)
  • Keep it concise: maxSpeed is better than theMaximumSpeedOfThePlayer
  • Use meaningful single letters for coordinates: x, y, w, h are acceptable for positions and dimensions
Good vs. Poor Variable Names
‚ùå Poor ‚úÖ Good Why?
var s = 100; var circleSize = 100; Descriptive name is clearer
var thing = 50; var eyeWidth = 50; Specific purpose is clear
var MYVAR = 200; var faceX = 200; Use camelCase, not all caps
var xyz123 = 75; var mouthHeight = 75; Meaningful beats random
When in doubt, imagine someone else will read your code in six months (or that YOU will read your code in six months after forgetting what it does). Choose variable names that will make sense without any additional context!

üî¢ Introduction to Data Types

So far, we've primarily worked with numbers in our variables. However, variables can store different types of data. Understanding data types helps you know what operations you can perform with your variables.

Common Data Types in JavaScript

Number
Numeric values, including integers (whole numbers) and decimals (floating-point numbers). Examples: 42, 3.14, -17, 0.5
var age = 25; var temperature = 98.6; var score = -10;
String
Text data enclosed in quotation marks (single or double quotes). Examples: "Hello", 'JavaScript', "123"
var playerName = "Alice"; var message = 'Welcome!'; var grade = "A+";
Boolean
True or false values, used for logic and decision-making. Only two possible values: true or false
var isGameOver = false; var playerWon = true; var hasKey = false;
JavaScript is "dynamically typed," which means you don't have to declare what type of data a variable will hold. The same variable can even hold different types at different times (though this isn't recommended for clarity). You'll learn more about data types and how to use them in upcoming lessons!

Type Matters for Operations

Different data types behave differently with operators:

Example 2.3.9: Numbers vs. Strings
// Numbers: addition works as expected var num1 = 5; var num2 = 3; var sum = num1 + num2; // Result: 8 // Strings: + concatenates (joins) them var word1 = "Hello"; var word2 = "World"; var phrase = word1 + word2; // Result: "HelloWorld" // Mixed: number becomes a string! var mixed = "Score: " + 100; // Result: "Score: 100"

Key Insight: The + operator behaves differently depending on the data types! With numbers, it adds. With strings, it joins them together. This is why choosing the right data type matters.

üé® Practical Applications

Building Flexible, Maintainable Drawings

Let's apply everything you've learned to create a sophisticated, easily-modifiable drawing:

Example 2.3.10: Complete Flexible Robot Drawing
// Position variables - change these to move the entire robot var robotX = 200; var robotY = 200; // Size variables - change these to resize the robot var headSize = 100; var bodyWidth = 80; var bodyHeight = 120; // Calculate other measurements based on sizes var eyeSize = headSize * 0.2; // Eyes are 20% of head var eyeSpacing = headSize * 0.3; // Eyes are 30% of head apart // Draw the robot // Head ellipse(robotX, robotY, headSize, headSize); // Eyes (positioned relative to robot center) ellipse(robotX - eyeSpacing, robotY - 10, eyeSize, eyeSize); // Left eye ellipse(robotX + eyeSpacing, robotY - 10, eyeSize, eyeSize); // Right eye // Body (positioned below head) rect(robotX - bodyWidth/2, robotY + headSize/2, bodyWidth, bodyHeight); // Antenna (positioned above head) line(robotX, robotY - headSize/2, robotX, robotY - headSize/2 - 30);

Benefits of This Approach:

  • Change robotX and robotY to move the entire robot instantly
  • Change headSize to resize the head‚Äîeyes automatically scale proportionally
  • All body parts remain properly positioned relative to each other
  • Easy to understand because variables have clear, descriptive names
  • Simple to modify and experiment with different designs
Professional programmers call this principle "DRY"‚ÄîDon't Repeat Yourself. By using variables, you avoid repeating the same values throughout your code, making it much easier to maintain and modify.
