üìê Section 1: Understanding the Canvas Coordinate System

What is a Canvas?

In p5.js (the JavaScript graphics library we'll be using), a canvas is your digital drawing surface‚Äîthink of it as a blank piece of paper on your screen. Every location on this canvas can be identified by two numbers: an x-coordinate (horizontal position) and a y-coordinate (vertical position).

Canvas

A rectangular area on the screen where graphics are drawn. The default canvas size in p5.js is 400 pixels wide by 400 pixels tall, but this can be customized using the createCanvas() function.

The Coordinate System: A Surprising Twist!

Here's something that surprises many new programmers: in computer graphics, the coordinate system is upside-down compared to what you learned in math class!

üìç Key Coordinate System Rules:
  • The origin point (0, 0) is at the top-left corner of the canvas
  • The x-coordinate increases as you move right (just like in math!)
  • The y-coordinate increases as you move down (opposite of math class!)
  • Coordinates are measured in pixels‚Äîthe tiny dots that make up your screen
Remember: Y increases downward! This is the opposite of the Cartesian coordinate system you learned in math. The point (100, 200) is 100 pixels from the left edge and 200 pixels from the TOP edge‚Äîputting it in the lower-left area of a 400√ó400 canvas.

Visualizing the Coordinate System

Let's look at some example coordinates on a 400√ó400 pixel canvas:

Point Coordinates (x, y) Location Description
(0, 0) x = 0, y = 0 Top-left corner
(400, 0) x = 400, y = 0 Top-right corner
(0, 400) x = 0, y = 400 Bottom-left corner
(400, 400) x = 400, y = 400 Bottom-right corner
(200, 200) x = 200, y = 200 Center of canvas
(100, 50) x = 100, y = 50 Near the top, slightly left of center
(300, 350) x = 300, y = 350 Near the bottom, right of center
When you're starting out, it helps to sketch your design on graph paper first, labeling key coordinates. This makes it much easier to translate your vision into code!

üé® Section 2: Drawing Basic Shapes

The ellipse() Function: Drawing Circles and Ovals

The ellipse() function is one of the most versatile drawing functions. It can create perfect circles or stretched ovals, depending on the parameters you provide.

ellipse(x, y, width, height)

Parameters:

  • x: x-coordinate of the ellipse's center
  • y: y-coordinate of the ellipse's center
  • width: horizontal diameter (in pixels)
  • height: vertical diameter (in pixels)

Note: When width and height are equal, you get a perfect circle!

üíª Code Examples: Drawing Ellipses

Example 1: Perfect Circle

ellipse(200, 200, 100, 100); // Circle at center with diameter 100

This creates a circle centered at (200, 200) with a diameter of 100 pixels.

Example 2: Horizontal Oval

ellipse(150, 100, 120, 60); // Wide oval near top-left

This creates an oval that's wider than it is tall, centered at (150, 100).

Example 3: Vertical Oval

ellipse(300, 300, 60, 120); // Tall oval near bottom-right

This creates an oval that's taller than it is wide, centered at (300, 300).

Remember: The ellipse is positioned by its CENTER point, not its top-left corner. This makes it easier to place circles exactly where you want them!

The rect() Function: Drawing Rectangles and Squares

The rect() function draws rectangles and squares. Unlike ellipses, rectangles are positioned by their top-left corner.

rect(x, y, width, height)

Parameters:

  • x: x-coordinate of the rectangle's top-left corner
  • y: y-coordinate of the rectangle's top-left corner
  • width: horizontal width (in pixels)
  • height: vertical height (in pixels)

Note: When width and height are equal, you get a perfect square!

üíª Code Examples: Drawing Rectangles

Example 1: Perfect Square

rect(150, 150, 100, 100); // Square with top-left at (150, 150)

This creates a 100√ó100 pixel square starting at position (150, 150).

Example 2: Horizontal Rectangle

rect(50, 50, 200, 80); // Wide rectangle near top-left

This creates a rectangle 200 pixels wide and 80 pixels tall.

Example 3: Vertical Rectangle

rect(300, 100, 60, 180); // Tall rectangle on right side

This creates a rectangle 60 pixels wide and 180 pixels tall.

Key Difference: Rectangles use their TOP-LEFT corner for positioning, while ellipses use their CENTER. Keep this in mind when planning your drawings!

The line() Function: Drawing Straight Lines

The line() function draws a straight line between two points. This is perfect for creating connections, geometric designs, or adding details to your drawings.

line(x1, y1, x2, y2)

Parameters:

  • x1: x-coordinate of the starting point
  • y1: y-coordinate of the starting point
  • x2: x-coordinate of the ending point
  • y2: y-coordinate of the ending point

Note: The order of points doesn't matter‚Äîthe line looks the same either way!

üíª Code Examples: Drawing Lines

Example 1: Horizontal Line

line(50, 200, 350, 200); // Horizontal line across middle

This draws a horizontal line from (50, 200) to (350, 200). Since y-coordinates are the same, the line is perfectly horizontal.

Example 2: Vertical Line

line(200, 50, 200, 350); // Vertical line down the center

This draws a vertical line from (200, 50) to (200, 350). Since x-coordinates are the same, the line is perfectly vertical.

Example 3: Diagonal Line

line(100, 100, 300, 300); // Diagonal line from top-left to bottom-right

This draws a diagonal line connecting the upper-left area to the lower-right area.

Combining Shapes: Creating a Simple Scene

The real power of these functions comes from combining them! Let's create a simple house:

üí¨ Section 3: Using Comments

What Are Comments?

Comments are notes that you write in your code that the computer completely ignores. They're for humans (including your future self!) to read and understand what the code is doing.

Comment

Text in your code that is ignored by the computer when the program runs. In JavaScript (and p5.js), comments start with two forward slashes: //

Everything after // on that line becomes a comment and won't be executed as code.

Why Use Comments?

Comments serve three crucial purposes in programming:

üìù Three Key Uses of Comments:
  1. Documentation: Explain what your code does and why you made certain choices
  2. Planning: Outline your program structure before writing the actual code
  3. Debugging: Temporarily disable code to test different parts of your program

Comment Syntax and Examples

üíª Comment Examples

Example 1: Documenting Code

// Draw the sun in the top-left corner ellipse(80, 80, 100, 100); // Draw the ground as a rectangle at the bottom rect(0, 300, 400, 100);

Example 2: Planning Before Coding

// TODO: Draw a snowman scene // 1. Sky background // 2. Snowy ground // 3. Snowman body (three circles) // 4. Face details (eyes, nose, mouth) // 5. Arms (lines)

Example 3: Commenting Out Code for Debugging

ellipse(200, 200, 100, 100); // Head // ellipse(175, 190, 20, 20); // Left eye (temporarily disabled) ellipse(225, 190, 20, 20); // Right eye

Best Practices for Writing Comments

Good comments explain WHY, not WHAT. Instead of writing // Draw a circle before ellipse(200, 200, 50, 50), write something more helpful like // Head of the snowman or // Sun in the sky.
‚úÖ Comment Best Practices:
  • Write comments that add value‚Äîexplain the purpose or reasoning behind the code
  • Keep comments concise and clear
  • Update comments when you change code‚Äîoutdated comments are worse than no comments!
  • Use comments to break your code into logical sections
  • Add a comment at the top of your program explaining what it does overall

Comments for Debugging

One of the most practical uses of comments is "commenting out" code‚Äîtemporarily disabling lines to test if they're causing problems. This is much faster than deleting code and retyping it!

üêõ Debugging with Comments

Scenario: Your drawing looks wrong, and you want to test if one shape is causing the problem.

Original Code:

ellipse(200, 200, 100, 100); // Head ellipse(175, 190, 20, 20); // Left eye ellipse(225, 190, 20, 20); // Right eye ellipse(200, 210, 30, 15); // Mouth (this might be wrong)

Commented Out for Testing:

ellipse(200, 200, 100, 100); // Head ellipse(175, 190, 20, 20); // Left eye ellipse(225, 190, 20, 20); // Right eye // ellipse(200, 210, 30, 15); // Mouth (disabled to test)

Now when you run the code, the mouth won't appear, helping you determine if it was positioned incorrectly.
