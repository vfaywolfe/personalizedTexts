Chapter 2.2: Variables and Data Types

Imagine trying to write a program where every number, every measurement, and every calculation must be hard-coded into your instructions. If you wanted to change the size of a shape, you'd have to hunt through your entire program to update every occurrence. This is where variables become one of the most powerful tools in programming.

Variables are like labeled containers that store information your program needs to remember. They allow you to write flexible, reusable code that can adapt to different situations. In this lesson, you'll discover how to use variables to make your programs more powerful, organized, and easier to modify. You'll learn how computer memory stores these values, and you'll practice using variables to create dynamic graphics that would be impossible with hard-coded values alone.

Key Concepts and Terms

Variable
A named location in computer memory that stores a value. The value can be retrieved later by referring to the variable's name. Variables allow programs to remember and reuse information.
Declaration
The act of creating a new variable and allocating memory for it. In JavaScript, we declare variables using the var keyword (or let and const in modern JavaScript).
Initialization
Assigning an initial value to a variable when it is declared. For example: var x = 10; both declares the variable x and initializes it with the value 10.
Assignment
Giving a variable a new value using the assignment operator =. After a variable is declared, you can change its value: x = 20;
Identifier
The name of a variable. Identifiers help programmers remember what information is stored in each variable. Good identifiers are descriptive and meaningful (e.g., faceSize instead of x).
Memory Diagram
A hand-drawn visualization showing how variables are stored in memory. Memory diagrams have two columns: the identifier (name) and the value (data), and help programmers trace code execution.
Expression
A combination of values, variables, and operators that evaluates to a single value. For example, x + y is an expression that adds the values of variables x and y.
Stack (Memory Stack)
The region of computer memory where variables are stored. When you declare a variable, space is allocated on the stack to hold its value.

Introduction to Variables

Variables are fundamental to programming. Think of a variable as a labeled box where you can store information. Just as you might label a box "Winter Clothes" to remember what's inside, you give variables meaningful names to remember what data they contain. The power of variables is that they let you write code once and use it in many different situations simply by changing the values stored in those variables.

Why Variables Matter

Without variables, you would need to hard-code every value in your program. Want to make all your circles bigger? You'd need to find and change every single number. Variables let you change one value in one place, and that change automatically applies everywhere the variable is used. This makes your code more flexible, easier to maintain, and more powerful.

Basic Variable Syntax

In JavaScript, you create a variable using the var keyword, followed by the name you want to give the variable, an equals sign, and the value you want to store: 

var myNumber = 42;
var faceSize = 150;
var eyeSize = 20;

Once you've declared a variable, you can use its name anywhere you would use a number: 

ellipse(200, 200, faceSize, faceSize); // draws circle with diameter from variable
Choosing Good Variable Names

Choose variable names that clearly describe what the variable represents. faceSize is much better than fs or x because it immediately tells you what the variable is for. Your code should read almost like English: "draw an ellipse with width faceSize and height faceSize."

Example 2.2.1: Creating Your First Variable
Step 1: Declare and initialize a variable

Let's create a variable to store the size of a circle:

var circleSize = 100;

This creates a variable named circleSize and stores the value 100 in it.

Step 2: Use the variable in a drawing command 
ellipse(200, 200, circleSize, circleSize);

The program looks up the value of circleSize (which is 100) and uses it for both the width and height parameters.

Result: A circle with diameter 100 pixels is drawn at position (200, 200). If you want to make the circle larger or smaller, you only need to change the value in the variable declaration‚Äîyou don't need to change the ellipse() command.

Using Variables in Drawing Programs

Variables become especially powerful when you use them in multiple places in your program. If you use a variable for the size of multiple shapes, changing that one variable will resize all those shapes at once. This is the key to writing flexible, maintainable code.

Example 2.2.2: Drawing a Face with Variables
Step 1: Declare variables for face and eye sizes 
var faceSize = 200;
var eyeSize = 30;
Step 2: Draw the face using the variable 
ellipse(200, 200, faceSize, faceSize); // face
Step 3: Draw the eyes using the eyeSize variable 
ellipse(170, 180, eyeSize, eyeSize); // left eye
ellipse(230, 180, eyeSize, eyeSize); // right eye
Result: A face with two eyes is drawn. Now, if you want a bigger face, just change faceSize to 300, and the face circle automatically becomes larger. If you want bigger eyes, change eyeSize to 40, and both eyes grow proportionally. This is much easier than finding and changing numbers throughout your code!
Common Mistake: Using Variables Before Declaration

You must declare a variable before you use it. If you try to use a variable that hasn't been declared yet, you'll get an error. Always put your var statements at the beginning of your program or at least before you try to use those variables. 

// WRONG - will cause an error:
ellipse(200, 200, size, size);
var size = 100;

// CORRECT:
var size = 100;
ellipse(200, 200, size, size);

Using Variables in Calculations

Variables can be used in mathematical expressions. The computer evaluates the expression and uses the result: 

var x = 100;
var y = 200;
var sum = x + y;  // sum now contains 300
println(sum);  // prints 300 to console
Example 2.2.3: Positioning Shapes with Variables
Step 1: Declare variables for positions 
var centerX = 200;
var centerY = 200;
var eyeOffset = 30;
Step 2: Use expressions with variables to position eyes 
// Left eye: subtract offset from center
ellipse(centerX - eyeOffset, centerY, 20, 20);

// Right eye: add offset to center
ellipse(centerX + eyeOffset, centerY, 20, 20);
Result: Two eyes are drawn, symmetrically positioned on either side of the center point. The left eye is at position (170, 200) and the right eye is at (230, 200). Now, if you want to move the entire face, just change centerX and centerY‚Äîboth eyes will automatically move with the face! If you want the eyes closer together or farther apart, just change eyeOffset.
Using Expressions for Relative Positioning

When you use expressions like centerX - eyeOffset, you're positioning elements relative to other elements. This is incredibly powerful because everything stays proportional and aligned no matter how you adjust the base values. This technique is used in professional programming to create responsive, adaptable interfaces.

Variables and Computer Memory

To truly understand variables, it helps to understand what's happening inside the computer. When you declare a variable, the computer sets aside a small piece of memory to store the value. Think of computer memory as a giant collection of numbered storage boxes. Each variable gets one of these boxes, and the variable name is like a label that helps you find the right box.

Understanding Memory Diagrams

Memory diagrams are hand-drawn tools that help programmers visualize what's happening in computer memory as code executes. They show which variables exist, what values they contain, and how those values change over time.

Memory diagrams are drawn as "inverted T" tables with two columns:

  • Identifier (left column): The name of the variable
  • Value (right column): The data stored in that variable

As you trace code from top to bottom, you add new variables to the memory diagram from bottom to top (new entries go above old ones). This might seem backwards at first, but it mimics how real computer memory works‚Äînew allocations happen "on top" of the existing memory stack.

Example 2.2.4: Creating a Memory Diagram
Given Code: 
/* 1 */ var x = 5;
/* 2 */ var y = 7;
/* 3 */ x = 9;
/* 4 */ var total = x + y;
/* 5 */ println(total);
Step 1: Trace line 1

The keyword var tells us to allocate new memory. Create a memory cell for x with value 5:

identifier value
x 5
Step 2: Trace line 2

Another var means another new variable. Add y with value 7 above x:

identifier value
y 7
x 5
Step 3: Trace line 3

No var keyword! This means we're updating an existing variable, not creating a new one. Cross out the old value and write the new one:

identifier value
y 7
x 5 9
Step 4: Trace line 4

New variable total. Evaluate the expression: x + y = 9 + 7 = 16. Add this to the diagram:

identifier value
total 16
y 7
x 5 9
Step 5: Trace line 5

The println statement doesn't create or modify any variables‚Äîit just outputs to the console. The memory diagram stays the same, and the program prints 16.

Final Result: The program prints 16 to the console. The memory diagram shows us exactly how the computer stored and updated values as the program ran. Notice how we can see the history of x's values (first 5, then 9) by looking at the crossed-out entries.
Reading Memory Diagrams

To find the current value of a variable, read the memory diagram from top to bottom and use the first (non-crossed-out) value you encounter for that variable. If a variable appears multiple times (which happens if you accidentally use var twice), the topmost entry is the current one‚Äîbut this usually indicates a programming mistake!

Example 2.2.5: The Danger of Re-declaring Variables
Code with a Common Mistake: 
/* 1 */ var x = 5;
/* 2 */ var y = 7;
/* 3 */ var x = 9;  // PROBLEM: var used again!
/* 4 */ var total = x + y;
/* 5 */ println(total);
Resulting Memory Diagram:

Line 3 uses var again, creating a second memory location also named x:

identifier value
total 16
x 9
y 7
x 5
What Happens: The program still prints 16 because we read from top to bottom and use the x with value 9. However, the original x with value 5 is now wasted memory (sometimes called "garbage"). The program can never access it again. This wastes computer resources and indicates poor programming practice. Lesson: Only use var when creating a brand new variable. To update an existing variable, just use the variable name without var.
Memory Diagram Best Practice

When you see the same variable name appear twice in a memory diagram, that's usually a sign of an error‚Äîyou probably used var when you shouldn't have. In professional code, this kind of mistake can cause hard-to-find bugs. Always be clear about whether you're creating a new variable or updating an existing one.

Real-World Applications and Best Practices

Understanding variables isn't just about drawing shapes‚Äîit's about learning to think like a programmer. Variables are used in every programming language and in every type of software, from video games to scientific simulations to mobile apps. Here are some best practices that will serve you well as you continue your programming journey:

Naming Conventions

Choose Meaningful Names

Your variable names should clearly describe what the variable represents. Compare these examples: 

// Bad: unclear what x and y represent
var x = 100;
var y = 30;

// Good: immediately clear from the names
var faceSize = 100;
var eyeSize = 30;

Good variable names make your code self-documenting. Anyone reading your code (including you, six months from now!) will immediately understand what each variable does.

Common Naming Styles

  • camelCase: Start with lowercase, capitalize each subsequent word: myVariableName, faceSize, leftEyePosition
  • Use descriptive names: rectangleWidth is better than rw
  • Avoid single letters except for very simple cases like loop counters or mathematical formulas
  • Be consistent: If you use Size for one variable, use it for others too: faceSize, eyeSize, mouthSize
Example 2.2.6: Refactoring Code to Use Variables
Original Code (Hard-Coded Values): 
ellipse(200, 200, 150, 150); // face
ellipse(170, 180, 30, 30); // left eye
ellipse(230, 180, 30, 30); // right eye
line(170, 240, 230, 240); // mouth

This code works, but it's inflexible. To resize the face, you'd need to carefully recalculate and change many numbers.

Improved Code (Using Variables): 
var centerX = 200;
var centerY = 200;
var faceSize = 150;
var eyeSize = 30;
var eyeOffset = 30;
var eyeHeight = -20;
var mouthY = 40;
var mouthWidth = 60;

// Face
ellipse(centerX, centerY, faceSize, faceSize);

// Eyes
ellipse(centerX - eyeOffset, centerY + eyeHeight, eyeSize, eyeSize);
ellipse(centerX + eyeOffset, centerY + eyeHeight, eyeSize, eyeSize);

// Mouth
line(centerX - mouthWidth/2, centerY + mouthY,
     centerX + mouthWidth/2, centerY + mouthY);
Benefits:
  • To move the entire face, just change centerX and centerY
  • To resize the face, change faceSize
  • To adjust eye spacing, modify eyeOffset
  • All proportions stay correct automatically
  • The code is more readable and maintainable
  • It's easy to create multiple faces with different sizes by changing just a few variables
Refactoring

The process of restructuring existing code to make it cleaner and more maintainable‚Äîwithout changing what it does‚Äîis called refactoring. Converting hard-coded numbers into well-named variables is one of the most common and valuable forms of refactoring. It's a skill professional programmers use every day to improve code quality.

Summary

In this lesson, you learned about one of the most fundamental concepts in programming: variables. Here are the key takeaways:

  • Variables are named storage locations in computer memory that hold values you want to remember and reuse in your programs.
  • Variables make code flexible: By using variables instead of hard-coded numbers, you can easily modify your program by changing values in one place.
  • Declaration vs. Assignment: Use var to declare a new variable, then use the variable name (without var) to update its value later.
  • Variables in expressions: You can use variables in mathematical expressions to create relationships between values in your program.
  • Memory diagrams help visualize how variables are stored and how their values change as your program executes.
  • Good naming practices: Choose descriptive, meaningful names in camelCase that clearly indicate what the variable represents.
  • Relative positioning: Using variables with expressions (like centerX + offset) lets you position elements relative to each other.
  • Refactoring: Converting hard-coded values into variables is a valuable technique for improving code quality and maintainability.
