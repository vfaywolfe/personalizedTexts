üåà Section 1: The RGB Color Model

How Computers Create Colors

Unlike mixing paints (where combining colors makes them darker), computer screens mix light. When you mix red, green, and blue light together at full intensity, you get white! This is called additive color mixing.

RGB Color Model

A way of representing colors using three numbers that specify the intensity of Red, Green, and Blue light. Each value ranges from 0 (no light/color) to 255 (maximum intensity).

Understanding RGB Values

Every RGB color is specified by three numbers in this format: (red, green, blue)

üî¢ RGB Value Ranges:
  • 0 = No light of that color (completely off)
  • 128 = Half intensity
  • 255 = Maximum intensity (completely on)

Common RGB Colors

Let's explore some fundamental colors and their RGB values:

Color RGB Values Explanation
Black (0, 0, 0) All lights off = black (absence of light)
White (255, 255, 255) All lights at maximum = white
Red (255, 0, 0) Full red, no green or blue
Green (0, 255, 0) Full green, no red or blue
Blue (0, 0, 255) Full blue, no red or green
Yellow (255, 255, 0) Red + Green = Yellow
Magenta (255, 0, 255) Red + Blue = Magenta
Cyan (0, 255, 255) Green + Blue = Cyan
Gray (128, 128, 128) Equal amounts of all three = neutral gray
Orange (255, 165, 0) Full red, some green, no blue
Purple (128, 0, 128) Half red, no green, half blue
Brown (165, 42, 42) More red, less green, less blue
Pattern Recognition: Notice that when all three RGB values are equal (like 0,0,0 or 128,128,128 or 255,255,255), you always get a shade of gray‚Äîfrom black to white!

Creating Custom Colors

The beauty of RGB is that you can create millions of unique colors by adjusting the three values. Here are some tips for mixing colors:

üé® Color Mixing Tips:
  • More red, less blue and green ‚Üí Warmer colors (orange, red, pink)
  • More blue, less red and green ‚Üí Cooler colors (blue, cyan, purple)
  • More green with some red ‚Üí Yellow, lime, olive tones
  • High values overall ‚Üí Lighter, brighter colors
  • Low values overall ‚Üí Darker, more muted colors
  • Similar but not equal values ‚Üí Subtle, sophisticated tones
Want to find specific RGB values? Search for "color picker" online, and you'll find tools that let you select colors visually and see their RGB values. Many design programs also include color pickers!

üñåÔ∏è Section 2: Applying Colors with Functions

The fill() Function: Coloring Shape Interiors

The fill() function sets the interior color for all shapes drawn after it. Think of it as choosing which paint to fill your shapes with.

fill(red, green, blue)

Sets the interior color for all subsequent shapes using RGB values.

Parameters:

  • red: Red component (0-255)
  • green: Green component (0-255)
  • blue: Blue component (0-255)

Important: The fill color persists‚Äîit affects ALL shapes drawn after it until you call fill() again with different values.

üíª fill() Examples

Example 1: Single Red Circle

fill(255, 0, 0); // Set fill to red ellipse(200, 200, 100, 100); // Draw a red circle

Example 2: Multiple Shapes with One Fill

fill(0, 0, 255); // Set fill to blue ellipse(100, 200, 50, 50); // Blue circle ellipse(200, 200, 50, 50); // Also blue ellipse(300, 200, 50, 50); // Also blue

Example 3: Changing Colors Between Shapes

fill(255, 0, 0); // Red ellipse(100, 200, 50, 50); // Red circle fill(0, 255, 0); // Green ellipse(200, 200, 50, 50); // Green circle fill(0, 0, 255); // Blue ellipse(300, 200, 50, 50); // Blue circle
Common Mistake: Remember that fill() affects ALL shapes after it, not just the next one. If you draw five circles after calling fill(255, 0, 0), all five will be red unless you call fill() again with different values.

The stroke() Function: Coloring Shape Outlines

The stroke() function sets the outline color for all shapes drawn after it. It works just like fill(), but affects the border instead of the interior.

stroke(red, green, blue)

Sets the outline color for all subsequent shapes using RGB values.

Parameters: Same as fill()‚Äîred, green, and blue values from 0-255.

üíª stroke() Examples

Example: Red Circle with Blue Outline

fill(255, 0, 0); // Red interior stroke(0, 0, 255); // Blue outline ellipse(200, 200, 100, 100); // Red circle with blue border

Example: Adjusting Stroke Weight

fill(255, 255, 0); // Yellow interior stroke(0, 0, 0); // Black outline strokeWeight(5); // Make outline thicker (5 pixels) rect(150, 150, 100, 100); // Yellow square with thick black border

The noStroke() Function: Removing Outlines

Sometimes you want shapes without any outline at all‚Äîthis creates a cleaner, more modern look. The noStroke() function removes outlines from all subsequent shapes.

noStroke()

Removes outlines from all subsequent shapes. Takes no parameters.

üíª noStroke() Example
noStroke(); // No outlines fill(255, 0, 0); // Red ellipse(150, 200, 80, 80); // Red circle, no outline fill(0, 255, 0); // Green ellipse(250, 200, 80, 80); // Green circle, no outline

The background() Function: Setting Canvas Color

The background() function sets the color of the entire canvas. It's typically called at the beginning of your program to establish the backdrop for your drawing.

background(red, green, blue)

Sets the background color of the entire canvas.

Parameters: RGB values from 0-255, just like fill() and stroke().

Note: The background() function fills the ENTIRE canvas, covering anything previously drawn.

üíª background() Examples

Example 1: Sky Blue Background

background(135, 206, 235); // Light blue sky color

Example 2: Black Background

background(0, 0, 0); // Black background

Example 3: Using Grayscale Shorthand

background(200); // Light gray (equivalent to 200, 200, 200)
Grayscale Shorthand: When you only provide one number to background(), fill(), or stroke(), it's interpreted as a gray value. For example, background(128) is the same as background(128, 128, 128)‚Äîmedium gray.

üñºÔ∏è Section 3: Creating a Complete Colored Scene

Building Complex Compositions

Now that you understand colors and shapes, let's combine everything to create a complete scene. The key to success is planning your composition and building it systematically, layer by layer.

Drawing Order Matters! In p5.js, shapes are drawn in the order they appear in your code. Later shapes appear "on top of" earlier shapes, like stacking pieces of paper. Plan your layers carefully!

Complete Example: Snowy Day with Snowman

Let's walk through creating a winter scene with a snowman, step by step. We'll start with the background and work our way forward, thinking about layers.

Step 1: Plan the layers (from back to front)
// Layer plan for our scene: // 1. Sky (background) // 2. Ground (white rectangle) // 3. Sun (yellow circle) // 4. Snowman body (three white circles) // 5. Snowman features (eyes, nose, buttons)
Step 2: Draw the sky background
background(135, 206, 235); // Sky blue background
Step 3: Draw the ground
noStroke(); // No outline needed fill(255, 255, 255); // White snow rect(0, 300, 400, 100); // Ground extending across canvas
Step 4: Draw the sun
fill(255, 255, 0); // Bright yellow ellipse(80, 64, 100, 100); // Sun in upper left
Step 5: Build the snowman (bottom to top)
// Snowman body (white circles, no stroke) fill(255, 255, 255); ellipse(200, 350, 150, 150); // Bottom ellipse(200, 250, 100, 100); // Middle ellipse(200, 170, 75, 75); // Head
Step 6: Add snowman features
// Eyes (black circles) fill(0, 0, 0); ellipse(190, 165, 8, 8); // Left eye ellipse(210, 165, 8, 8); // Right eye // Nose (orange triangle - using a thin rectangle for simplicity) fill(255, 140, 0); // Orange rect(198, 175, 4, 15); // Buttons (black circles) fill(0, 0, 0); ellipse(200, 240, 10, 10); ellipse(200, 260, 10, 10); ellipse(200, 280, 10, 10);
Complete Code:
// Sunny Snowy Day Scene // Sky background background(135, 206, 235); // Ground noStroke(); fill(255, 255, 255); rect(0, 300, 400, 100); // Sun fill(255, 255, 0); ellipse(80, 64, 100, 100); // Snowman body fill(255, 255, 255); ellipse(200, 350, 150, 150); // Bottom ellipse(200, 250, 100, 100); // Middle ellipse(200, 170, 75, 75); // Head // Eyes fill(0, 0, 0); ellipse(190, 165, 8, 8); ellipse(210, 165, 8, 8); // Nose fill(255, 140, 0); rect(198, 175, 4, 15); // Buttons fill(0, 0, 0); ellipse(200, 240, 10, 10); ellipse(200, 260, 10, 10); ellipse(200, 280, 10, 10);
Notice the workflow: We started with the background, added major elements, then added details. We used noStroke() to create clean shapes without outlines, and we carefully changed fill() colors before each element. This systematic approach prevents errors and makes complex scenes manageable!
‚úèÔ∏è Formative Assessment 2.2.3: Complete Scenes

Question 1 (Multiple Choice):

In what order are shapes drawn in a p5.js program?

Show Answer

B) In the order they appear in the code, from top to bottom - Shapes drawn later appear on top of shapes drawn earlier, like layering paper.

Question 2 (True/False):

You should call background() at the end of your program after drawing all shapes.

Show Answer

False. You should call background() at the BEGINNING of your program because it fills the entire canvas and would cover any shapes already drawn.

Question 3 (Code Analysis):

What will be wrong with this code?

fill(255, 0, 0); ellipse(200, 200, 100, 100); background(0, 0, 255); rect(150, 150, 50, 50);
Show Answer

A) The circle will be invisible - Calling background() in the middle of your code covers everything drawn before it. The circle is drawn first, then covered by the blue background. Only the rectangle (drawn after background) will be visible.

Question 4 (Planning):

If you're creating a scene with a house, trees, and clouds, in what order should these elements be drawn? Why?

Show Sample Answer

Sample Answer: First draw the background (sky), then clouds (behind everything), then trees (behind house), then the house (in front). This creates proper depth‚Äîthings in the back are drawn first, things in front are drawn last. Each layer covers parts of the previous layers, creating a realistic sense of depth.

Question 5 (Code Writing Challenge):

Write code to draw a simple traffic light with: a dark gray rectangular post, and three circles (red, yellow, green) stacked vertically inside. Add appropriate colors and a sky blue background.

Show Sample Solution
// Traffic light scene background(135, 206, 235); // Sky blue // Post (dark gray) noStroke(); fill(64, 64, 64); rect(180, 100, 40, 200); // Red light (top) fill(255, 0, 0); ellipse(200, 130, 30, 30); // Yellow light (middle) fill(255, 255, 0); ellipse(200, 180, 30, 30); // Green light (bottom) fill(0, 255, 0); ellipse(200, 230, 30, 30);

Note: Your solution may vary in positions and sizes. The key is having the background, post, and three appropriately colored circles!
