üìê Section 1: The Canvas Coordinate System

‚è±Ô∏è ~15 minutes

Understanding the Canvas (El Lienzo)

Think of the programming canvas like a digital fabric you're working with in costume design. Just as you measure fabric from the top-left corner when cutting patterns, our canvas also starts at the top-left corner!

üì± Example: TikTok Video Layout

When you create TikTok videos or Instagram posts, the app uses a coordinate system just like this! When you tap to add text or stickers:

  • X-coordinate (horizontal) determines left-to-right position
  • Y-coordinate (vertical) determines top-to-bottom position

The same principle applies to programming graphics!

Key Concepts

Canvas (lienzo)

The drawing surface where all visual elements appear. By default, it's 400 pixels wide and 400 pixels tall.

Coordinates (coordenadas)

A pair of numbers (x, y) that specify an exact location on the canvas. Think of it like coordinates on a patient chart - they tell you exactly where something is located.

Origin Point (0, 0)

The starting point at the top-left corner of the canvas. This is different from math class where (0, 0) is at the bottom-left!

The coordinate system might seem backwards at first (y increases going DOWN instead of UP), but you'll get used to it quickly. Many students find it helpful to sketch designs on paper first with labeled coordinates before coding them.

Coordinate Examples

Location Coordinates (x, y) Description
Top-left corner (0, 0) Starting point
Top-right corner (400, 0) Far right, top edge
Bottom-left corner (0, 400) Far bottom, left edge
Bottom-right corner (400, 400) Maximum coordinates
Center (200, 200) Middle of canvas
üíÑ Example: Beauty Tutorial Layout

Imagine you're creating graphics for a beauty tutorial video:

  • Product name at (200, 50) - centered near the top
  • Tutorial steps at (50, 150) - left side, below the title
  • Your profile picture at (350, 350) - bottom right corner

The coordinates tell the computer exactly where to place each element, just like you'd plan where to position text overlays on a video!

‚≠ï Section 2: Drawing Basic Shapes

‚è±Ô∏è ~20 minutes

Now that you understand the coordinate system, let's start drawing! We'll use three fundamental shape functions.

2.1 Drawing Circles and Ovals: ellipse()

ellipse(x, y, width, height)

Parameters:

  • x - horizontal position of the CENTER
  • y - vertical position of the CENTER
  • width - how wide the circle/oval is
  • height - how tall the circle/oval is

Note: If width = height, you get a perfect circle!

In medical software, ellipse() might be used to:

  • Draw blood cells in microscopy visualizations
  • Create circular buttons for vital signs monitors
  • Represent pills or medication in dosage apps
  • Show pulse or blood pressure indicators

Code Examples

// Perfect circle (ancho = altura) ellipse(200, 200, 100, 100); // Circle at center, 100 pixels wide/tall // Oval shape (horizontal) ellipse(150, 100, 150, 80); // Wide oval // Oval shape (vertical) ellipse(300, 250, 60, 120); // Tall oval
üé≠ Example: Costume Design - Button Details

Imagine you're creating a digital design for costume buttons:

// Large decorative button for a period costume ellipse(200, 200, 80, 80); // Main button (perfect circle) // Smaller buttons for a shirt collar ellipse(150, 100, 30, 30); // Left button ellipse(250, 100, 30, 30); // Right button // Oval buttons for a jacket ellipse(200, 300, 50, 70); // Elongated button

2.2 Drawing Rectangles and Squares: rect()

rect(x, y, width, height)

Parameters:

  • x - horizontal position of the TOP-LEFT corner
  • y - vertical position of the TOP-LEFT corner
  • width - how wide the rectangle is
  • height - how tall the rectangle is

Key difference from ellipse: positioned by top-left corner, not center!

Notice that ellipse() uses the CENTER point, but rect() uses the TOP-LEFT corner. This is one of the most common sources of confusion for beginners. Don't worry if you mix these up at first - everyone does!
üì± Example: Creating a Phone Screen Mockup

Let's design elements like you'd see in a social media app:

This creates a basic layout similar to Instagram or TikTok!

2.3 Drawing Lines: line()

line(x1, y1, x2, y2)

Draws a straight line between two points.

Parameters:

  • x1, y1 - starting point coordinates
  • x2, y2 - ending point coordinates
Medical applications use line() to create:

  • üìà ECG/EKG heartbeat traces
  • üå°Ô∏è Temperature trend lines on patient charts

üí¨ Section 3: Code Comments

‚è±Ô∏è ~10 minutes

Comments are notes you write in your code to explain what you're doing. Think of them like notes you'd write in your nursing textbook margins - they help you remember why you did something!

Comment (comentario)

Text in your code that the computer ignores. Created with two forward slashes //

Purpose: Document your code, explain complex logic, temporarily disable code

How to Write Comments

// This is a single-line comment // The computer completely ignores this text ellipse(200, 200, 100, 100); // Draw a circle in the center // You can "comment out" code to temporarily disable it: // rect(50, 50, 100, 100); // This rectangle won't appear /* This is a multi-line comment that can span multiple lines. Useful for longer explanations! */
Comments in code are just like patient documentation in nursing:

  • üìã Patient charts explain WHY certain treatments were given
  • üíä Medication notes record important details for the next shift
  • üè• Code comments explain WHY you wrote code a certain way

Both help the next person (or future you!) understand what happened and why.

Good vs. Bad Comments

‚ùå Not Helpful - States the Obvious
// Draw a circle ellipse(200, 200, 100, 100); // Make a rectangle rect(50, 50, 80, 80);

These comments just repeat what the code already says clearly.

‚úÖ Helpful - Explains Purpose and Context
// Medication pill visualization - circular shape represents tablet ellipse(200, 200, 100, 100); // Patient chart background - white rectangle for vital signs display rect(50, 50, 80, 80);

These comments explain the PURPOSE and provide context.

Write comments that answer "WHY?" not "WHAT?". The code itself shows what you're doing - comments should explain your reasoning or the purpose behind it. This is especially helpful when you return to your code later!

Using Comments for Debugging

One powerful use of comments is temporarily "turning off" code to find problems:

// Testing which shape is causing the problem ellipse(200, 100, 80, 80); // This one works fine // rect(150, 200, 100, 50); // Temporarily disabled to test line(100, 300, 300, 300); // This one works fine

üé® Section 4: Adding Color with RGB

‚è±Ô∏è ~15 minutes

Now let's add color to your creations! Just like mixing colors for costume designs or makeup, we'll mix digital colors using the RGB system.

RGB (Rojo, Verde, Azul)

A color system that mixes Red, Green, and Blue light.

Each color component ranges from 0 to 255:

  • 0 = none of that color
  • 255 = maximum intensity of that color
üíÑ Example: RGB Like Makeup Mixing

Think of RGB like mixing makeup or paint colors:

  • Red eyeshadow = RGB(255, 0, 0) - pure red, no green or blue
  • Soft pink blush = RGB(255, 192, 203) - lots of red, some green/blue for softness
  • Teal eyeliner = RGB(0, 128, 128) - medium green and blue, no red
  • White highlight = RGB(255, 255, 255) - maximum of all colors
  • Black mascara = RGB(0, 0, 0) - absence of all colors

Common RGB Colors

Color Name RGB Values Example Use
üî¥ Red (255, 0, 0) Emergency alerts, stop signs
üü¢ Green (0, 255, 0) Normal vital signs, go signals
üîµ Blue (0, 0, 255) Information, trust
‚ö™ White (255, 255, 255) Cleanliness, medical environments
‚ö´ Black (0, 0, 0) Text, outlines
üíõ Yellow (255, 255, 0) Caution warnings
üíú Purple (128, 0, 128) Branding, creativity
üß° Orange (255, 165, 0) Moderate alerts, warmth
ü©∑ Pink (255, 192, 203) Soft, friendly interfaces

Color Functions

fill(red, green, blue)

Sets the interior color for shapes. Affects ALL shapes drawn after it until you call fill() again.

stroke(red, green, blue)

Sets the outline color for shapes. Also affects lines!

noStroke()

Removes outlines from shapes. Useful for creating smooth, filled shapes without borders.

background(red, green, blue)

Sets the canvas background color. Usually called first in your code.

In healthcare software, colors communicate vital information:

  • üî¥ Red (255, 0, 0) - Critical alerts, abnormal vital signs
  • üü° Yellow (255, 255, 0) - Warnings, moderate concerns
  • üü¢ Green (0, 255, 0) - Normal values, stable patient
  • üîµ Blue (0, 150, 255) - Information, patient data
  • ‚ö™ White (255, 255, 255) - Background, clean interface

You'll use these same RGB principles when creating health visualizations!

Color Code Examples

// Set sky blue background (cielo azul) background(135, 206, 235); // Draw a red heart (coraz√≥n rojo) - no outline noStroke(); fill(255, 0, 0); ellipse(180, 200, 80, 80); // Left lobe ellipse(220, 200, 80, 80); // Right lobe // Draw green grass (c√©sped verde) fill(34, 139, 34); rect(0, 350, 400, 50); // Draw a sun (sol amarillo) with orange outline stroke(255, 140, 0); // Orange outline fill(255, 255, 0); // Yellow fill ellipse(350, 50, 80, 80);
Colors "stick" until you change them! Once you call fill(255, 0, 0), ALL shapes after that will be red until you call fill() again with different values. This is one of the most common mistakes beginners make.

üé≠ Section 5: Creating Complete Visual Compositions

‚è±Ô∏è ~20 minutes

Important Concept: Drawing Order Matters

Shapes are drawn in the order they appear in your code, like layers in costume design or digital art. Later shapes appear ON TOP of earlier shapes.
Think of it like costume layering:
  • First layer: Base garment (drawn first, appears on bottom)
  • Second layer: Vest or jacket (drawn second, covers the base)
  • Third layer: Accessories (drawn last, appear on top)

In code, the same principle applies!

Let's create a simple patient vital signs monitor display:

// Patient Monitor Display - Vital Signs Visualization // Step 1: White background (clean medical interface) background(255, 255, 255); // Step 2: Monitor screen (dark background for display) fill(20, 20, 40); noStroke(); rect(50, 50, 300, 250); // Step 3: Heart rate zone - GREEN (normal range) fill(0, 200, 0); rect(70, 80, 260, 60); // Step 4: Heart icon - RED fill(255, 0, 0); ellipse(100, 110, 30, 30); // Heart shape simplified // Step 5: Heart rate text area - WHITE fill(255, 255, 255); rect(140, 95, 180, 30); // Step 6: Blood pressure zone - BLUE (stable) fill(0, 150, 255); rect(70, 160, 260, 60); // Step 7: Temperature zone - YELLOW (normal) fill(255, 200, 0); rect(70, 240, 260, 40); // This creates a simple multi-zone vital signs display // just like you'd see in a hospital!

Notice how we build from back to front - background first, then each display zone layer by layer.
